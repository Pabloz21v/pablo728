Sesión 2 — Diseño de la base de datos

Objetivo: Diseñar y crear la tabla productos, poblarla y practicar consultas básicas.

Preparación

  • MySQL Workbench abierto por cada alumno.
  • Breve explicación del uso de la herramienta.

Paso a paso

  1. Teoría rápida
    • Repasar conceptos previos de SQL.
  2. Crear tabla productos

    SQL sugerido:

    CREATE TABLE productos (
      id INT AUTO_INCREMENT PRIMARY KEY,
      codigo VARCHAR(30) UNIQUE,
      nombre VARCHAR(120) NOT NULL,
      descripcion TEXT,
      categoria VARCHAR(60),
      cantidad INT NOT NULL DEFAULT 0,
      precio DECIMAL(10,2) NOT NULL DEFAULT 0.00,
      fecha_ingreso DATE NOT NULL
    );

    Ejecutar en Workbench.

  3. Insertar datos de prueba

    Insertar 2 filas (ejemplo):

    INSERT INTO productos (codigo, nombre, descripcion, categoria, cantidad, precio, fecha_ingreso)
    VALUES
      ('SKU001','Tornillo M4','Paquete 100','Ferreteria',200,0.05,'2025-09-01'),
      ('SKU002','Monitor 24"','LED 1080p','Electronica',10,129.99,'2025-07-15');

    Este código es de ejemplo.

  4. Práctica
    • Inventar su propia tabla, aplicar conocimientos de SQL para insertar valores.

Ejercicios prácticos

  • Hacer backup exportando la tabla SQL que personalizaron y guardarla en sus correos, pendrive, etc.

Errores comunes

  • Tipos DECIMAL: usar DECIMAL(10,2) para evitar redondeos inesperados.
  • Fecha: formato YYYY-MM-DD.
