Sesión 14: Mecanismos de Infección y Prevención

3.1 Vectores de Ataque

Los vectores de ataque son los métodos o rutas que utilizan los cibercriminales para acceder a sistemas, redes o datos. Comprender estos vectores es fundamental para implementar defensas efectivas.

3.1.1 Correos Electrónicos

El correo electrónico sigue siendo el vector de ataque más común, utilizado en aproximadamente el 90% de los ciberataques exitosos.

Tipos de ataques por correo electrónico:
  • Phishing: Correos que suplantan identidades para robar credenciales
  • Spear Phishing: Ataques dirigidos a individuos específicos
  • Whaling: Ataques dirigidos a ejecutivos de alto nivel
  • Business Email Compromise (BEC): Comprometimiento de cuentas corporativas

Técnicas comunes en ataques por correo:

  • Archivos adjuntos maliciosos:
    • Documentos con macros (Word, Excel, PowerPoint)
    • Archivos ejecutables disfrazados
    • PDFs con JavaScript malicioso
    • Archivos comprimidos con malware
  • Enlaces maliciosos:
    • Redirección a sitios de phishing
    • Descarga automática de malware
    • Explotación de vulnerabilidades del navegador
  • Ingeniería social:
    • Creación de urgencia o miedo
    • Suplantación de autoridades
    • Ofertas demasiado buenas para ser verdad

3.1.2 Descargas de Internet

Las descargas desde Internet representan otro vector crítico de infección.

Tipo de Descarga Riesgo Ejemplos Medidas Preventivas
Software pirata Muy Alto Cracks, keygens, versiones modificadas Usar solo software legítimo
Freeware/Shareware Medio Software gratuito con adware Descargar de fuentes oficiales
Actualizaciones falsas Alto Updates de Flash, Java, navegadores Actualizar solo desde sitios oficiales
Torrents/P2P Muy Alto Archivos compartidos infectados Evitar redes P2P para software

Drive-by Downloads:

Descargas automáticas que ocurren sin conocimiento del usuario al visitar sitios web comprometidos:

  • Explotación de vulnerabilidades del navegador
  • Plugins desactualizados (Flash, Java)
  • Scripts maliciosos en sitios legítimos comprometidos
  • Anuncios maliciosos (malvertising)

3.1.3 Dispositivos USB y Medios Removibles

Los dispositivos de almacenamiento removible siguen siendo un vector de ataque significativo, especialmente en entornos corporativos.

Riesgos de dispositivos USB:
  • Autorun malicioso: Ejecución automática de malware
  • Archivos infectados: Propagación de virus tradicionales
  • BadUSB: Firmware comprometido que simula otros dispositivos
  • Rubber Ducky: Dispositivos que simulan teclados para ejecutar comandos

Técnicas de ataque USB:

  1. USB Drop Attack: Dejar USBs infectados en lugares públicos
  2. Regalos corporativos: USBs promocionales infectados
  3. Dispositivos perdidos: USBs "encontrados" que contienen malware
  4. Ingeniería social: Convencer a empleados de usar USBs específicos

3.1.4 Redes Sociales

Las plataformas de redes sociales se han convertido en vectores de ataque cada vez más populares.

Métodos de ataque en redes sociales:

  • Perfiles falsos: Creación de identidades falsas para ganar confianza
  • Enlaces maliciosos: Compartir links que redirigen a sitios peligrosos
  • Aplicaciones maliciosas: Apps que solicitan permisos excesivos
  • Información personal: Recopilación de datos para ataques dirigidos
  • Catfishing: Creación de relaciones falsas para obtener información
Información valiosa para atacantes en redes sociales:
  • Fechas de nacimiento y lugares
  • Nombres de familiares y mascotas
  • Lugares de trabajo y posiciones
  • Ubicaciones frecuentadas
  • Intereses y aficiones
  • Contactos profesionales y personales

3.2 Técnicas de Prevención

3.2.1 Firewalls

Los firewalls actúan como una barrera entre redes confiables y no confiables, controlando el tráfico entrante y saliente basándose en reglas de seguridad predeterminadas.

Tipos de Firewalls:

Tipo Funcionamiento Ventajas Limitaciones
Packet Filtering Examina paquetes individuales Rápido, eficiente No examina contenido
Stateful Mantiene estado de conexiones Más seguro que packet filtering Mayor consumo de recursos
Application Layer Examina contenido de aplicaciones Control granular Puede impactar rendimiento
Next Generation Combina múltiples tecnologías Protección integral Complejo de configurar

Configuración básica de firewall:

  • Denegar todo por defecto (default deny)
  • Permitir solo tráfico necesario
  • Configurar reglas para tráfico entrante y saliente
  • Monitorear logs regularmente
  • Actualizar reglas según necesidades del negocio

3.2.2 Software Antivirus

El software antivirus utiliza múltiples técnicas para detectar, prevenir y eliminar malware.

Métodos de detección:

  1. Detección basada en firmas:
    • Compara archivos con base de datos de firmas conocidas
    • Efectivo contra malware conocido
    • Requiere actualizaciones constantes
  2. Detección heurística:
    • Analiza comportamiento sospechoso
    • Puede detectar variantes de malware conocido
    • Mayor tasa de falsos positivos
  3. Detección basada en comportamiento:
    • Monitorea acciones en tiempo real
    • Detecta malware zero-day
    • Puede impactar rendimiento del sistema
  4. Machine Learning:
    • Utiliza algoritmos de IA para detección
    • Mejora con el tiempo
    • Efectivo contra amenazas nuevas
Mejores prácticas para antivirus:
  • Mantener definiciones de virus actualizadas
  • Realizar escaneos completos regularmente
  • Configurar protección en tiempo real
  • No instalar múltiples antivirus simultáneamente
  • Complementar con otras herramientas de seguridad

3.2.3 Actualizaciones de Software

Mantener el software actualizado es una de las medidas de seguridad más importantes y efectivas.

Tipos de actualizaciones:

  • Parches de seguridad: Corrigen vulnerabilidades específicas
  • Actualizaciones de funcionalidad: Añaden nuevas características
  • Actualizaciones críticas: Solucionan problemas graves de seguridad
  • Service Packs: Colecciones grandes de actualizaciones

Estrategia de actualización:

  1. Inventario de software: Mantener lista de todo el software instalado
  2. Priorización: Actualizar primero software crítico y expuesto
  3. Pruebas: Probar actualizaciones en entorno de desarrollo
  4. Automatización: Configurar actualizaciones automáticas cuando sea apropiado
  5. Monitoreo: Verificar que las actualizaciones se apliquen correctamente

3.2.4 Copias de Seguridad (Backups)

Las copias de seguridad son esenciales para la recuperación ante incidentes de seguridad, especialmente ataques de ransomware.

Regla 3-2-1 para backups:
  • 3 copias de datos importantes
  • 2 en diferentes tipos de medios
  • 1 fuera del sitio (offsite)

Tipos de backup:

Tipo Descripción Tiempo de Backup Tiempo de Restauración
Completo Copia todos los datos Largo Rápido
Incremental Solo cambios desde último backup Rápido Lento
Diferencial Cambios desde último backup completo Medio Medio
Snapshot Imagen instantánea del sistema Muy rápido Muy rápido

3.3 Buenas Prácticas para Usuarios y Empresas

3.3.1 Buenas Prácticas para Usuarios

Gestión de Contraseñas:

  • Usar contraseñas únicas para cada cuenta
  • Crear contraseñas largas y complejas (mínimo 12 caracteres)
  • Utilizar gestores de contraseñas
  • Habilitar autenticación de dos factores (2FA) cuando esté disponible
  • Cambiar contraseñas predeterminadas inmediatamente

Navegación Segura:

  • Verificar la legitimidad de sitios web antes de ingresar información
  • Buscar el candado SSL (https://) en sitios que manejan información sensible
  • Evitar hacer clic en enlaces sospechosos
  • Usar navegadores actualizados con configuraciones de seguridad apropiadas
  • Instalar bloqueadores de anuncios para prevenir malvertising

Manejo de Correo Electrónico:

  • Verificar la identidad del remitente antes de abrir adjuntos
  • Sospechar de correos que crean urgencia o miedo
  • No proporcionar información personal por correo
  • Usar filtros de spam y phishing
  • Reportar correos sospechosos al departamento de IT
Señales de alerta en correos electrónicos:
  • Errores ortográficos y gramaticales
  • Saludos genéricos ("Estimado cliente")
  • Solicitudes urgentes de información personal
  • Enlaces que no coinciden con el dominio legítimo
  • Amenazas de cierre de cuenta

3.3.2 Buenas Prácticas para Empresas

Políticas de Seguridad:

  • Desarrollar y mantener políticas de seguridad claras
  • Realizar capacitaciones regulares de concienciación en seguridad
  • Implementar principios de menor privilegio
  • Establecer procedimientos de respuesta a incidentes
  • Realizar auditorías de seguridad periódicas

Gestión de Acceso:

  • Implementar control de acceso basado en roles (RBAC)
  • Revisar periódicamente permisos de usuarios
  • Deshabilitar cuentas de empleados que ya no trabajen en la empresa
  • Usar cuentas administrativas solo cuando sea necesario
  • Monitorear actividades de usuarios privilegiados

Segmentación de Red:

  • Separar redes críticas de redes de usuarios
  • Implementar VLANs para diferentes departamentos
  • Aislar sistemas de control industrial
  • Crear redes separadas para invitados
  • Monitorear tráfico entre segmentos

Gestión de Dispositivos:

  • Mantener inventario actualizado de todos los dispositivos
  • Implementar políticas de dispositivos móviles (MDM)
  • Configurar dispositivos con configuraciones de seguridad estándar
  • Encriptar datos en dispositivos móviles y laptops
  • Implementar capacidad de borrado remoto
Marco de Ciberseguridad NIST:
  1. Identificar: Comprender el entorno de ciberseguridad
  2. Proteger: Implementar salvaguardas apropiadas
  3. Detectar: Identificar ocurrencia de eventos de ciberseguridad
  4. Responder: Tomar acción ante incidentes detectados
  5. Recuperar: Restaurar servicios y capacidades
