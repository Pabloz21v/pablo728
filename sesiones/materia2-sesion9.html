2. Relaciones 1 a 1 con FOREIGN KEY

Concepto:

Una relación 1 a 1 significa que cada registro en una tabla tiene un solo registro relacionado en otra tabla.

Ejemplo práctico:

  • Tabla clientes y tabla direcciones.
  • Cada cliente tiene una sola dirección, y cada dirección pertenece a un solo cliente.

FOREIGN KEY (Clave Foránea)

Establece una relación entre dos tablas, garantizando la integridad referencial.

Función: Asegura que el valor en una columna de una tabla corresponda a un valor existente en la tabla referenciada.

Sintaxis básica: FOREIGN KEY (columna_local) REFERENCES tabla_referencia(columna_referencia)

Interpretación : "Esta columna debe contener valores que existan en otra tabla específica."

REFERENCES (Referencias)

Especifica qué tabla y columna son referenciadas por la clave foránea.

Función: Define el destino de la relación, indicando la tabla padre y su columna primary key.

Sintaxis: REFERENCES nombre_tabla(nombre_columna)

Interpretación : "Los valores de esta columna deben existir en la columna especificada de la tabla indicada."

ON DELETE CASCADE (Eliminación en Cascada)

Define qué sucede con los registros relacionados cuando se elimina el registro padre.

Función: Elimina automáticamente todos los registros hijos cuando se elimina el registro padre.

Alternativas:

  • ON DELETE SET NULL: Establece NULL en la clave foránea
  • ON DELETE RESTRICT: Impide la eliminación si existen registros relacionados
  • ON DELETE NO ACTION: Similar a RESTRICT

Interpretación : "Si elimino un cliente, también elimina automáticamente su dirección."

Script para crear las tablas:

-- Tabla principal: clientes
CREATE TABLE clientes (
    id_cliente INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    fecha_registro DATE NOT NULL
);

-- Tabla relacionada: direcciones
CREATE TABLE direcciones (
    id_direccion INT AUTO_INCREMENT PRIMARY KEY,
    calle VARCHAR(100) NOT NULL,
    ciudad VARCHAR(50) NOT NULL,
    codigo_postal VARCHAR(20),
    id_cliente INT UNIQUE,
    FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente) ON DELETE CASCADE
);#INTERPRETACION DE:
#FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente) ON DELETE CASCADE
#establecer como clave foranea la columna id_cliente de la tabla direcciones
referenciandola con la columna id_cliente de la tabla clientes y ante la eliminacion
de un cliente se borran los registros en la tabla direcciones relacionados al id_cliente..
Interpretación : "Cada cliente tiene una dirección única, y si se elimina un cliente, su dirección también se elimina automáticamente (ON DELETE CASCADE)."

3. Tipos de Datos: DECIMAL y DATE

DECIMAL(p,s)

  • Uso: Números exactos, como precios o medidas.
  • p: Número total de dígitos.
  • s: Número de decimales.
  • Ejemplo: DECIMAL(10,2) → 12345678.90 (8 dígitos enteros y 2 decimales).

DATE

  • Uso: Almacena fechas en formato YYYY-MM-DD.
  • YYYY = año. 
  • MM = mes.
  • DD = dia.
  • Ejemplo: '2025-10-27'.
  • Ejemplo: '2000-07-30'.
  • Ejemplo: '1989-02-15'.

ALTER TABLE (Alterar Tabla)

Modifica la estructura de una tabla existente.

Función: Permite agregar, eliminar o modificar columnas y restricciones.

Sintaxis básica: ALTER TABLE nombre_tabla [ADD|DROP|MODIFY|CHANGE] ...

Interpretación : "Quiero cambiar algo en la estructura de esta tabla que ya existe."

ADD COLUMN (Agregar Columna)

Agrega una nueva columna a una tabla existente.

Función: Permite expandir la estructura de datos sin perder información existente.

Sintaxis: ALTER TABLE tabla ADD COLUMN nombre_columna tipo_dato [restricciones]

Interpretación : "Agregar un nuevo campo a esta tabla para almacenar información adicional."

DEFAULT (Valor por Defecto)

Establece un valor automático para una columna cuando no se proporciona un valor explícito.

Función: Garantiza que la columna tenga un valor incluso si no se especifica durante la inserción.

Sintaxis: nombre_columna tipo_dato DEFAULT valor_por_defecto

Interpretación : "Si no especifico un valor para este campo, usa automáticamente este valor predeterminado."

Modificación de la tabla clientes para incluir DECIMAL:

ALTER TABLE clientes
ADD COLUMN saldo DECIMAL(10,2) DEFAULT 0.00;#modificar la tabla clientes agregando una columna llamada saldo con
el tipo de dato DECIMAL(10,2) y un valor por defecto automatico 0.00
Interpretación : "Ahora cada cliente tendrá un campo /columna `saldo` que almacena valores monetarios con 2 decimales, como $100.50."

4. Insertar Valores NULL

Formas de insertar NULL:

1. Omitir la columna (si permite NULL):

INSERT INTO clientes (nombre, email, fecha_registro)
VALUES ('Juan Pérez', 'juan@example.com', '2025-09-01');

(El campo `saldo` se llenará con el valor por defecto: `0.00`)

2. Insertar explícitamente NULL:

INSERT INTO clientes (nombre, email, fecha_registro, saldo)
VALUES ('Ana Gómez', 'ana@example.com', '2025-09-02', NULL);
#Suele dar errores por lo que no lo recomiendo

(El campo `saldo` quedará como `NULL` si no tiene valor por defecto)

5. Modificar el Nombre y Tipo de Dato de una Columna

CHANGE COLUMN (Cambiar Columna)

Cambia tanto el nombre como el tipo de dato de una columna existente.

Función: Permite renombrar una columna y modificar su definición completamente.

Sintaxis: ALTER TABLE tabla CHANGE COLUMN nombre_actual nombre_nuevo tipo_dato

Interpretación : "Quiero cambiar el nombre de esta columna y posiblemente también su tipo de dato."

MODIFY COLUMN (Modificar Columna)

Cambia únicamente el tipo de dato de una columna, manteniendo su nombre.

Función: Permite alterar las características de una columna sin cambiar su nombre.

Sintaxis: ALTER TABLE tabla MODIFY COLUMN nombre_columna nuevo_tipo_dato

Interpretación : "Mantener el nombre de la columna, pero cambiar su tipo de dato o restricciones."

Cambiar el nombre de una columna:

ALTER TABLE clientes
CHANGE COLUMN email correo_electronico VARCHAR(100);#modificar en la tabla clientes el nombre de la columna email por 
correo_electronico y varchar de 100 caracteres.
Interpretación : "La columna `email` ahora se llamará `correo_electronico`."

Cambiar el tipo de dato de una columna:

ALTER TABLE clientes
MODIFY COLUMN fecha_registro DATETIME;#cambiar en la tabla clientes el tipo de dato de la 
columna fecha_registro a DATETIME
Interpretación : "El campo `fecha_registro` ahora almacenará fecha y hora (ej: `2025-09-01 14:30:00`)."

7. Ejemplos Prácticos 

LEFT JOIN (Unión Izquierda)

Propósito: Combina registros de dos tablas, incluyendo todos los registros de la tabla izquierda.

Función: Muestra todos los registros de la primera tabla, incluso si no tienen coincidencias en la segunda tabla.

Sintaxis: SELECT ... FROM tabla1 LEFT JOIN tabla2 ON tabla1.columna = tabla2.columna

Ejemplo: SELECT * FROM clientes LEFT JOIN direcciones ON clientes.id_clientes= direcciones.id_clientes

Interpretación : "Muestra todos los clientes, incluso aquellos que no tienen dirección registrada."

Abreviación de Nombres de Tabla (Alias)

Crear nombres cortos para las tablas dentro de una consulta.

Función: Simplifica la escritura de consultas complejas y mejora la legibilidad.

Sintaxis: FROM tabla_nombre AS alias o simplemente FROM tabla_nombre alias

Ejemplo: FROM clientes c donde 'c' es el alias para 'clientes'

Ejemplo2: FROM direcciones d donde 'd' es el alias para 'direcciones'

Interpretación : "En lugar de escribir 'clientes' cada vez, usaré simplemente 'c'."

SELECT FROM WHERE con Funciones de Fecha

YEAR(): Extrae el año de una fecha

MONTH(): Extrae el mes de una fecha (1-12)

DAY(): Extrae el día del mes de una fecha (1-31)

Filtrar y analizar datos basándose en componentes específicos de fechas.

Sintaxis: WHERE YEAR(fecha_columna) = 2025

Interpretación : "Mostrar solo los registros del año 2025" o "Filtrar por mes de septiembre"


Ejemplo 1: Relación 1 a 1 usando LEFT JOIN y alias

-- Mostrar el cliente y su dirección
SELECT c.nombre, d.calle, d.ciudad
FROM clientes c
LEFT JOIN direcciones d ON c.id_cliente = d.id_cliente;#selecciona la columna nombre de la tabla clientes y,
calle y ciudad de la tabla direcciones donde la tabla clientes
(alias de clientes con la letra c) relacionandolo con la tabla direcciones
(alias de direcciones con la letra d) en donde la columa 
id_clientes de la tabla clientes sea igual a la columna 
clientes de la tabla direcciones
Interpretación : "Muestra el nombre de cada cliente junto a su calle y ciudad. Si no tiene dirección, aparecerá `NULL`. Uso 'c' como alias para 'clientes' y 'd' para 'direcciones' para simplificar la escritura."

Ejemplo 2: Filtrar por NULL

-- Clientes sin dirección
SELECT c.nombre
FROM clientes c
LEFT JOIN direcciones d ON c.id_cliente = d.id_cliente
WHERE d.calle IS NULL;#selecciona la columna nombre de la tabla clientes(abreviando clientes como c)
relacionandolo con la tabla direcciones (abreviando direcciones como d)
en donde la columna id_cliente de la tabla cliente sea igual a 
la columna id_cliente de la tabla direcciones donde en la tabla direciones 
la columna calle es null
Interpretación : "Lista los clientes que no tienen una calle registrada."

Ejemplo 3: Usar DECIMAL

-- Clientes con saldo mayor a 100
SELECT nombre, saldo
FROM clientes
WHERE saldo > 100;#selecciona la columna nombre y saldo de la tabla clientes donde el saldo sea mayor a 100
Interpretación : "Muestra los clientes que tienen un saldo superior a $100.00."

Ejemplo 4: Usar DATE con funciones YEAR, MONTH, DAY

-- Clientes registrados en septiembre de 2025
SELECT nombre, fecha_registro
FROM clientes
WHERE YEAR(fecha_registro) = 2025 AND MONTH(fecha_registro) = 9;#seleciona las columnas nombre y fecha_registro de la tabla clientes donde
el año en la columna fecha_registro sea igual a 2025 y el mes 
en la columa fecha_registro sea igual a 9
Interpretación : "Lista los clientes que se registraron en septiembre de 2025."

Ejemplo 5: Combinando funciones de fecha

-- Clientes registrados el día 1 de cualquier mes
SELECT nombre, fecha_registro
FROM clientes c
WHERE DAY(c.fecha_registro) = 1;#seleciona las columnas nombre y fecha_registro de la tabla clientes
(abreviado con la letra c) donde 
el dia en la columna fecha_registro de la tabla clientes 
sea igual a 1
Interpretación : "Muestra los clientes que se registraron el primer día de cualquier mes."

9. Resumen y Cierre

Conceptos Clave Aprendidos:

  • Relaciones 1 a 1: Usan FOREIGN KEY, REFERENCES y UNIQUE.
  • NULL: Se inserta omitiendo el valor, usando NULL explícitamente, o dejando campos vacíos.
  • DECIMAL: Ideal para valores monetarios y números exactos.
  • DATE: Almacena fechas sin hora en formato YYYY-MM-DD.
  • Modificar tablas: ALTER TABLE + CHANGE o MODIFY.
  • LEFT JOIN: Combina tablas mostrando todos los registros de la tabla izquierda. 
  • Alias: Simplifican las consultas usando nombres cortos para las tablas.
  • Funciones de fecha: YEAR(), MONTH(), DAY() para filtrar por componentes de fecha.
