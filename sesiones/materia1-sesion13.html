Sesión 13: Virus Informáticos: Tipos y Funcionamiento

2.1 Definición y Clasificación de Virus Informáticos

Virus Informático: Un programa malicioso que se replica insertando copias de sí mismo en otros programas, archivos o sectores de arranque del disco duro. Un virus requiere un programa anfitrión para propagarse y ejecutarse.

Los virus informáticos se pueden clasificar según varios criterios:

Por su Ubicación en el Sistema

  • Virus de archivos: Infectan archivos ejecutables (.exe, .com)
  • Virus de boot/arranque: Infectan el sector de arranque del disco
  • Virus de macro: Infectan documentos con macros (Word, Excel)
  • Virus residentes: Permanecen en memoria RAM
  • Virus no residentes: Se ejecutan solo cuando se activa el archivo infectado

Por su Comportamiento

  • Virus destructivos: Diseñados para dañar o eliminar datos
  • Virus no destructivos: Se replican sin causar daño inmediato
  • Virus furtivos: Intentan ocultarse de la detección
  • Virus polimórficos: Cambian su código para evitar detección
  • Virus metamórficos: Reescriben completamente su código

2.2 Tipos de Malware

Virus

Como se definió anteriormente, requieren un programa anfitrión para replicarse y propagarse.

Características de los Virus:
  • Necesitan un archivo anfitrión
  • Se replican automáticamente
  • Pueden permanecer latentes
  • Su activación puede ser por fecha, evento o acción específica

Gusanos (Worms)

Programas maliciosos que se replican automáticamente a través de redes sin necesidad de un archivo anfitrión.

Característica Virus Gusano
Archivo anfitrión Requerido No requerido
Propagación Manual (ejecutar archivo infectado) Automática por red
Objetivo principal Dañar archivos/sistema Consumir recursos de red
Detección Más difícil Más fácil (tráfico de red)

Troyanos (Trojans)

Programas que aparentan ser legítimos pero contienen código malicioso oculto. No se replican por sí mismos.

Tipos de Troyanos:

  • RAT (Remote Access Trojan): Permite control remoto del sistema
  • Banking Trojans: Roban información financiera
  • Backdoor Trojans: Crean puertas traseras en el sistema
  • DDoS Trojans: Convierten el sistema en parte de una botnet
  • Downloader Trojans: Descargan otros malwares

Spyware

Software diseñado para recopilar información del usuario sin su conocimiento o consentimiento.

Tipos de información que recopila el Spyware:
  • Historial de navegación
  • Credenciales de acceso
  • Información personal y financiera
  • Pulsaciones de teclado (keyloggers)
  • Capturas de pantalla

Adware

Software que muestra publicidad no deseada, generalmente en forma de ventanas emergentes o banners.

Características del Adware:

  • Muestra anuncios excesivos
  • Puede ralentizar el sistema
  • A menudo se instala junto con software gratuito
  • Puede recopilar datos de navegación
  • Algunos tipos son más agresivos que otros

Rootkits

Colección de software malicioso diseñado para obtener control administrativo sobre un sistema mientras permanece oculto.

Tipo de Rootkit Ubicación Dificultad de Detección Persistencia
Rootkits de aplicación Nivel de aplicación Baja Baja
Rootkits de kernel Núcleo del sistema Alta Alta
Rootkits de bootkit Sector de arranque Muy alta Muy alta
Rootkits de firmware BIOS/UEFI Extrema Extrema

2.3 Mecanismos de Propagación y Daños Comunes

Mecanismos de Propagación

1. Medios de almacenamiento removibles

  • USB, CD/DVD, discos externos
  • Autorun/Autoplay en Windows
  • Archivos infectados copiados manualmente

2. Correo electrónico

  • Archivos adjuntos maliciosos
  • Enlaces a sitios infectados
  • Imágenes con código embebido
  • Documentos con macros maliciosas

3. Descargas de Internet

  • Software pirateado o crackeado
  • Actualizaciones falsas
  • Plugins y extensiones maliciosas
  • Sitios web comprometidos

4. Redes

  • Explotación de vulnerabilidades de red
  • Recursos compartidos sin protección
  • Conexiones P2P
  • Redes Wi-Fi inseguras

Daños Comunes

Daños a Nivel de Sistema:
  • Corrupción o eliminación de archivos
  • Modificación del registro de Windows
  • Desactivación de software de seguridad
  • Ralentización significativa del sistema
  • Inestabilidad y bloqueos frecuentes

Daños a Nivel de Datos:

  • Robo de información personal y financiera
  • Cifrado de archivos (ransomware)
  • Modificación no autorizada de documentos
  • Pérdida de datos críticos
  • Compromiso de credenciales de acceso

Daños a Nivel de Red:

  • Consumo excesivo de ancho de banda
  • Ataques DDoS desde el sistema infectado
  • Propagación a otros sistemas de la red
  • Comprometimiento de la seguridad de red

2.4 Ejemplos Reales y Casos de Estudio Recientes

Caso de Estudio 1: WannaCry (2017)

Tipo: Ransomware con capacidades de gusano

Impacto: Infectó más de 300,000 computadoras en 150 países

Mecanismo de propagación:

  • Explotó la vulnerabilidad EternalBlue en Windows
  • Se propagó automáticamente por redes sin intervención del usuario
  • Cifraba archivos y exigía pago en Bitcoin

Sectores afectados:

  • Sistema Nacional de Salud del Reino Unido
  • FedEx y otras grandes corporaciones
  • Sistemas ferroviarios en Alemania
  • Hospitales y sistemas críticos mundialmente

Caso de Estudio 2: Emotet (2014-2021)

Tipo: Troyano bancario evolucionado a botnet

Impacto: Considerado una de las botnets más peligrosas del mundo

Evolución de Emotet:
  • Fase 1: Troyano bancario simple
  • Fase 2: Downloader para otros malwares
  • Fase 3: Plataforma completa de Malware-as-a-Service

Técnicas utilizadas:

  • Correos de phishing altamente sofisticados
  • Documentos de Office con macros maliciosas
  • Capacidades de auto-propagación en redes locales
  • Actualizaciones modulares
  • Técnicas de evasión de detección

Caso de Estudio 3: SolarWinds (2020)

Tipo: Ataque de cadena de suministro / Backdoor

Impacto: Comprometió múltiples agencias gubernamentales y empresas Fortune 500

Metodología del ataque:

  1. Compromiso de la infraestructura de compilación de SolarWinds
  2. Inserción de backdoor (SUNBURST) en actualizaciones legítimas
  3. Distribución del malware a través del proceso normal de actualizaciones
  4. Activación selectiva solo en objetivos de alto valor
  5. Movimiento lateral dentro de las redes comprometidas
Lecciones aprendidas:
  • Importancia de la seguridad en la cadena de suministro
  • Necesidad de monitoreo continuo de actividades de red
  • Implementación de principios de Zero Trust
  • Verificación de integridad de software

Tendencias Actuales en Malware (2023-2024)

1. Malware-as-a-Service (MaaS)

  • Comercialización del malware
  • Reducción de barreras de entrada para cibercriminales
  • Modelos de suscripción y soporte técnico

2. Ataques dirigidos a la nube

  • Compromiso de cuentas de servicios en la nube
  • Explotación de configuraciones incorrectas
  • Cryptojacking en recursos de nube

3. Inteligencia Artificial en malware

  • Evasión automatizada de sistemas de detección
  • Generación automática de variantes
  • Optimización de campañas de phishing

4. Ataques a dispositivos IoT

  • Botnets de dispositivos IoT
  • Ataques DDoS masivos
  • Compromiso de privacidad en hogares inteligentes
