Sesión 15: Protección de Datos Personales y Empresariales

4.1 ¿Qué son los Datos Personales y por qué Protegerlos?

Datos Personales: Cualquier información relativa a una persona física identificada o identificable. Una persona identificable es aquella cuya identidad puede determinarse, directa o indirectamente, mediante un identificador como un nombre, un número de identificación, datos de localización, un identificador en línea o uno o varios elementos propios de la identidad física, fisiológica, genética, psíquica, económica, cultural o social de dicha persona.

4.1.1 Categorías de Datos Personales

Categoría Ejemplos Nivel de Protección Regulaciones Aplicables
Datos de Identificación Nombre, apellidos, DNI, dirección Estándar RGPD, LOPD
Datos Financieros Número de tarjeta, cuentas bancarias Alto PCI DSS, RGPD
Datos de Salud Historial médico, diagnósticos Muy Alto HIPAA, RGPD
Datos Biométricos Huellas dactilares, reconocimiento facial Muy Alto RGPD, regulaciones locales
Datos de Comportamiento Historial de navegación, compras Medio RGPD, CCPA

4.1.2 Importancia de la Protección de Datos

Riesgos para Individuos:

  • Robo de identidad: Uso fraudulento de información personal para cometer delitos
  • Fraude financiero: Acceso no autorizado a cuentas bancarias y tarjetas de crédito
  • Chantaje y extorsión: Uso de información comprometedora para coaccionar
  • Pérdida de privacidad: Exposición de información íntima o personal
  • Discriminación: Uso de datos para tomar decisiones discriminatorias
  • Acoso y stalking: Uso de información personal para acosar

Riesgos para Empresas:

  • Multas regulatorias: Sanciones por incumplimiento de normativas
  • Pérdida de confianza: Daño a la reputación y pérdida de clientes
  • Costos de recuperación: Gastos en investigación, notificación y remediación
  • Responsabilidad legal: Demandas y litigios por parte de afectados
  • Pérdida competitiva: Exposición de secretos comerciales
  • Interrupción operacional: Impacto en las operaciones comerciales
Estadísticas sobre violaciones de datos:
  • El 83% de las organizaciones han experimentado más de una violación de datos
  • Se necesitan en promedio 287 días para identificar y contener una violación
  • El costo promedio por registro perdido es de $150
  • El 95% de las violaciones exitosas se deben a error humano

4.2 Técnicas de Cifrado y Anonimización

4.2.1 Cifrado (Encryption)

El cifrado es el proceso de convertir información legible (texto plano) en información codificada (texto cifrado) que solo puede ser decodificada por aquellos que poseen la clave correcta.

Tipos de Cifrado:

Cifrado Simétrico

Utiliza la misma clave tanto para cifrar como para descifrar datos.

  • Ventajas: Rápido, eficiente para grandes volúmenes de datos
  • Desventajas: Problema de distribución de claves
  • Algoritmos comunes: AES, DES, 3DES, Blowfish
  • Usos típicos: Cifrado de archivos, bases de datos, comunicaciones en tiempo real
Cifrado Asimétrico

Utiliza un par de claves relacionadas matemáticamente: una pública y una privada.

  • Ventajas: Soluciona el problema de distribución de claves
  • Desventajas: Más lento que el cifrado simétrico
  • Algoritmos comunes: RSA, ECC, Diffie-Hellman
  • Usos típicos: Intercambio de claves, firmas digitales, certificados
Algoritmo Tipo Tamaño de Clave Seguridad Rendimiento
AES Simétrico 128, 192, 256 bits Muy Alta Excelente
RSA Asimétrico 1024-4096 bits Alta Lento
ECC Asimétrico 160-571 bits Muy Alta Moderado
ChaCha20 Simétrico 256 bits Muy Alta Excelente

4.2.2 Implementación de Cifrado

Cifrado en Tránsito:

  • TLS/SSL: Protege datos durante transmisión web (HTTPS)
  • VPN: Cifra todo el tráfico de red entre puntos
  • SFTP/SCP: Transferencia segura de archivos
  • Email encryption: S/MIME, PGP para correos electrónicos

Cifrado en Reposo:

  • Cifrado de disco completo: BitLocker, FileVault, LUKS
  • Cifrado de archivos: AxCrypt, 7-Zip con contraseña
  • Cifrado de base de datos: TDE (Transparent Data Encryption)
  • Cifrado de backups: Proteger copias de seguridad
Mejores Prácticas de Cifrado:
  • Usar algoritmos de cifrado estándar y actuales
  • Implementar gestión segura de claves
  • Cifrar datos tanto en tránsito como en reposo
  • Realizar rotación regular de claves
  • Mantener copias seguras de claves de recuperación
  • Usar longitudes de clave apropiadas

4.2.3 Anonimización de Datos

La anonimización es el proceso de eliminar o modificar información personal de manera que los individuos no puedan ser identificados directa o indirectamente.

Técnicas de Anonimización:

Eliminación de Identificadores
  • Remover nombres, direcciones, números de identificación
  • Eliminar fechas específicas
  • Quitar información de contacto
Generalización
  • Reemplazar valores específicos con rangos (edad 25 → 20-30)
  • Usar categorías más amplias (código postal completo → ciudad)
  • Agrupar valores similares
Supresión
  • Eliminar completamente ciertos campos
  • Remover registros que son únicos o identificables
  • Suprimir valores atípicos
Perturbación
  • Agregar ruido estadístico a los datos
  • Redondear valores numéricos
  • Intercambiar valores entre registros similares
Técnica Utilidad de Datos Nivel de Protección Casos de Uso
K-anonimity Media Medio Investigación médica
L-diversity Media-Baja Alto Datos sensibles
T-closeness Baja Muy Alto Datos muy sensibles
Differential Privacy Variable Muy Alto Análisis estadístico

4.3 Herramientas para Proteger Datos

4.3.1 VPN (Virtual Private Networks)

Una VPN crea una conexión segura y encriptada entre un dispositivo y una red privada a través de Internet.

Funcionamiento de VPN: La VPN encripta el tráfico del usuario y lo enruta a través de un servidor VPN antes de llegar a su destino final. Esto oculta la dirección IP real del usuario y cifra todos los datos transmitidos.

Tipos de VPN:

VPN de Acceso Remoto
  • Permite a usuarios individuales conectarse a una red corporativa
  • Ideal para teletrabajo
  • Protocolos comunes: OpenVPN, IKEv2, WireGuard
VPN Site-to-Site
  • Conecta redes completas entre sí
  • Utilizada para conectar oficinas remotas
  • Más compleja de configurar pero más escalable
VPN de Consumidor
  • Servicios comerciales para usuarios individuales
  • Protege privacidad en redes públicas
  • Permite eludir restricciones geográficas

Beneficios de las VPN:

  • Cifrado de tráfico de red
  • Ocultación de dirección IP
  • Acceso seguro a recursos remotos
  • Protección en Wi-Fi públicos
  • Bypass de censura y restricciones

4.3.2 Gestores de Contraseñas

Los gestores de contraseñas almacenan y generan contraseñas seguras para múltiples cuentas.

Características de un buen gestor de contraseñas:
  • Cifrado fuerte de la base de datos de contraseñas
  • Generación de contraseñas aleatorias y seguras
  • Sincronización segura entre dispositivos
  • Autenticación de dos factores
  • Auditoría de seguridad de contraseñas
  • Autocompletado seguro en navegadores

4.3.3 Autenticación Multifactor (MFA)

La autenticación multifactor requiere dos o más factores de verificación para conceder acceso a un recurso.

Factores de autenticación:

  1. Algo que sabes: Contraseñas, PINs, preguntas de seguridad
  2. Algo que tienes: Teléfonos móviles, tokens, tarjetas inteligentes
  3. Algo que eres: Biometría (huellas, rostro, voz, iris)
  4. Algo que haces: Patrones de comportamiento, firma
  5. Dónde estás: Geolocalización, direcciones IP

Métodos comunes de MFA:

  • SMS: Códigos enviados por mensaje de texto
  • Aplicaciones autenticadoras: Google Authenticator, Authy
  • Tokens de hardware: YubiKey, RSA SecurID
  • Notificaciones push: Aprobación a través de apps móviles
  • Biometría: Huellas dactilares, reconocimiento facial
  • Códigos de respaldo: Códigos únicos para recuperación
Mejores Prácticas para MFA:
  • Preferir aplicaciones autenticadoras sobre SMS
  • Mantener códigos de respaldo en lugar seguro
  • Usar tokens de hardware para cuentas críticas
  • Implementar MFA en todas las cuentas importantes
  • Educar a usuarios sobre la importancia de MFA

4.4 Seguridad en la Nube y Almacenamiento Seguro

4.4.1 Modelos de Servicio en la Nube

Modelo Descripción Responsabilidades del Cliente Ejemplos
IaaS Infraestructura como Servicio OS, aplicaciones, datos, identidad AWS EC2, Azure VMs
PaaS Plataforma como Servicio Aplicaciones, datos, identidad Google App Engine, Heroku
SaaS Software como Servicio Datos, identidad, dispositivos Office 365, Salesforce

4.4.2 Responsabilidades Compartidas en la Nube

El modelo de responsabilidad compartida define qué aspectos de la seguridad son responsabilidad del proveedor de nube y cuáles del cliente.

Responsabilidades del Proveedor de Nube:

  • Seguridad física de los centros de datos
  • Seguridad de la infraestructura de red
  • Seguridad del hardware y software base
  • Disponibilidad y redundancia de servicios
  • Cumplimiento de certificaciones de seguridad

Responsabilidades del Cliente:

  • Gestión de identidades y accesos
  • Configuración segura de servicios
  • Cifrado de datos
  • Monitoreo y logging
  • Gestión de vulnerabilidades en aplicaciones

4.4.3 Mejores Prácticas para Seguridad en la Nube

Gestión de Identidades:

  • Implementar principio de menor privilegio
  • Usar autenticación multifactor para todos los usuarios
  • Revisar periódicamente permisos y accesos
  • Implementar single sign-on (SSO) cuando sea posible
  • Monitorear actividades de usuarios privilegiados

Cifrado de Datos:

  • Cifrar datos tanto en tránsito como en reposo
  • Usar claves de cifrado gestionadas por el cliente cuando sea posible
  • Implementar cifrado a nivel de aplicación para datos sensibles
  • Rotar claves de cifrado regularmente

Configuración Segura:

  • Deshabilitar servicios y puertos innecesarios
  • Configurar correctamente grupos de seguridad y firewalls
  • Implementar logging y monitoreo completo
  • Usar herramientas de escaneo de configuración
  • Mantener inventario actualizado de recursos
Herramientas de Seguridad en la Nube:
  • CASB: Cloud Access Security Brokers para visibilidad y control
  • CWPP: Cloud Workload Protection Platforms para servidores
  • CSPM: Cloud Security Posture Management para configuración
  • SIEM: Security Information and Event Management para monitoreo

4.4.4 Compliance y Certificaciones

Principales estándares de seguridad en la nube:

  • SOC 2: Controles de seguridad, disponibilidad y confidencialidad
  • ISO 27001: Sistema de gestión de seguridad de la información
  • PCI DSS: Seguridad de datos de tarjetas de pago
  • HIPAA: Protección de información de salud
  • GDPR: Regulación general de protección de datos
