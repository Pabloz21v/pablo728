Sesión 16: Seguridad en Dispositivos y Respuesta a Incidentes

5.1 Seguridad en Dispositivos Móviles y Computadoras

5.1.1 Seguridad en Dispositivos Móviles

Los dispositivos móviles presentan desafíos únicos de seguridad debido a su portabilidad, conectividad constante y el almacenamiento de gran cantidad de información personal y empresarial.

Principales amenazas a dispositivos móviles:
  • Malware móvil: Aplicaciones maliciosas, troyanos, spyware
  • Pérdida o robo: Acceso físico no autorizado al dispositivo
  • Redes Wi-Fi inseguras: Interceptación de datos en redes públicas
  • Aplicaciones maliciosas: Apps con permisos excesivos o código malicioso
  • Phishing móvil: SMS y aplicaciones de phishing
  • Man-in-the-Middle: Interceptación de comunicaciones

Mejores Prácticas para Seguridad Móvil:

Configuración del Dispositivo
  • Bloqueo de pantalla: PIN, patrón, contraseña, biometría
  • Cifrado del dispositivo: Activar cifrado completo del almacenamiento
  • Autobloqueo: Configurar bloqueo automático tras inactividad
  • Actualizaciones automáticas: Mantener OS y apps actualizadas
  • Ubicación remota: Activar servicios de localización para recuperación
Gestión de Aplicaciones
  • Descargar apps solo de tiendas oficiales (Google Play, App Store)
  • Revisar permisos antes de instalar aplicaciones
  • Desinstalar apps no utilizadas regularmente
  • Leer reseñas y verificar desarrollador antes de instalar
  • Evitar hacer "jailbreak" o "root" al dispositivo
Plataforma Características de Seguridad Herramientas Nativas Consideraciones Especiales
iOS Sandboxing, App Store curado, cifrado fuerte Find My, Screen Time, Keychain Ecosistema cerrado, mayor control
Android Google Play Protect, sandboxing, permisos granulares Find My Device, Google Play Protect Fragmentación, mayor personalización

5.1.2 Seguridad en Computadoras

Las computadoras de escritorio y portátiles requieren múltiples capas de protección debido a su complejidad y las múltiples vías de ataque disponibles.

Seguridad del Sistema Operativo
  • Actualizaciones regulares: Instalar parches de seguridad inmediatamente
  • Control de cuentas de usuario (UAC): Configurar niveles apropiados
  • Firewall: Activar y configurar correctamente
  • Antivirus/EDR: Instalar solución de seguridad robusta
  • Cifrado de disco: BitLocker, FileVault, o equivalentes
Configuración de Navegadores
  • Mantener navegadores actualizados
  • Configurar actualizaciones automáticas de plugins
  • Desactivar plugins innecesarios (Flash, Java en navegador)
  • Usar bloqueadores de anuncios y scripts
  • Configurar navegación privada para sitios sensibles
  • Gestionar cookies y datos de sitios web
Configuraciones de Seguridad Esenciales en Windows:
  • Windows Defender habilitado y actualizado
  • SmartScreen activado
  • Control de aplicaciones y navegador
  • Protección contra ransomware activada
  • BitLocker para cifrado de disco
  • Windows Update configurado correctamente
Gestión de Software
  • Mantener inventario de software instalado
  • Desinstalar programas no utilizados
  • Usar administradores de paquetes cuando sea posible
  • Verificar integridad de descargas (checksums)
  • Evitar software pirata o de fuentes no confiables

5.2 Detección de Infecciones y Brechas de Seguridad

5.2.1 Indicadores de Compromiso (IoC)

Los indicadores de compromiso son evidencias forenses que sugieren que un sistema ha sido comprometido por actividad maliciosa.

Indicadores de Nivel de Sistema:

  • Rendimiento degradado: Lentitud inusual del sistema
  • Alto uso de recursos: CPU, memoria o red sin explicación
  • Procesos desconocidos: Servicios o aplicaciones no reconocidas
  • Archivos modificados: Cambios inesperados en archivos del sistema
  • Conexiones de red sospechosas: Tráfico hacia IPs desconocidas
  • Cambios en configuración: Modificaciones no autorizadas

Indicadores de Nivel de Red:

  • Tráfico de red inusual o en horarios no habituales
  • Conexiones a dominios o IPs maliciosas conocidas
  • Protocolos de comunicación inusuales
  • Gran volumen de datos salientes
  • Patrones de tráfico anómalos
Señales de Alerta Inmediata:
  • Pantallas de ransomware
  • Archivos cifrados con extensiones extrañas
  • Mensajes de antivirus sobre detecciones
  • Cambios inesperados en página de inicio del navegador
  • Ventanas emergentes excesivas
  • Usuarios reportando problemas similares

5.2.2 Técnicas de Detección

Monitoreo Proactivo:

  • Análisis de logs: Revisión regular de registros del sistema
  • Monitoreo de red: Análisis de tráfico en tiempo real
  • Análisis de comportamiento: Detección de anomalías
  • Threat Hunting: Búsqueda proactiva de amenazas
  • Honeypots: Sistemas señuelo para detectar ataques

Análisis Forense Digital:

  1. Preservación: Crear copias bit-a-bit de evidencias
  2. Adquisición: Recopilar datos de múltiples fuentes
  3. Análisis: Examinar evidencias para encontrar indicadores
  4. Documentación: Registrar hallazgos y procedimientos
  5. Presentación: Preparar informes y evidencias

5.3.2 Análisis de Logs del Sistema

Los logs del sistema contienen información valiosa sobre actividades normales y anómalas que pueden indicar compromisos de seguridad.

Tipos de Logs Importantes:

  • Logs de autenticación: Intentos de login exitosos y fallidos
  • Logs de sistema: Arranque, apagado, errores del sistema
  • Logs de aplicaciones: Eventos específicos de aplicaciones
  • Logs de seguridad: Cambios de permisos, accesos a recursos
  • Logs de firewall: Tráfico permitido y bloqueado
  • Logs de proxy/web: Actividad de navegación

Herramientas de Análisis de Logs:

Windows
  • Event Viewer: Herramienta nativa de Windows
  • PowerShell: Scripts para análisis automatizado
  • Sysmon: Logging avanzado de actividad del sistema
  • WinLogBeat: Exportación de logs a sistemas centralizados
Linux
  • rsyslog/journald: Sistemas nativos de logging
  • grep/awk/sed: Herramientas de línea de comandos
  • Logwatch: Análisis automatizado de logs
  • ELK Stack: Elasticsearch, Logstash, Kibana
Eventos de Windows Critical para Seguridad:
  • 4624: Logon exitoso
  • 4625: Logon fallido
  • 4648: Logon usando credenciales explícitas
  • 4720: Cuenta de usuario creada
  • 4732: Usuario agregado a grupo privilegiado
  • 7045: Servicio instalado

5.3.3 Herramientas de Análisis de Red

Analizadores de Tráfico:

  • Wireshark: Captura y análisis detallado de paquetes
  • tcpdump: Herramienta de línea de comandos para captura
  • NetworkMiner: Análisis forense de tráfico de red
  • Zeek (Bro): Monitor de seguridad de red

Herramientas de Monitoreo:

  • Nagios: Monitoreo de infraestructura
  • PRTG: Monitoreo de red y ancho de banda
  • SolarWinds: Suite completa de monitoreo
  • Splunk: Análisis de logs y datos de máquina

5.4 Pasos a Seguir ante un Incidente de Seguridad

5.4.1 Plan de Respuesta a Incidentes

Un plan de respuesta a incidentes es un conjunto documentado de procedimientos para detectar, responder y recuperarse de incidentes de ciberseguridad.

Incidente de Seguridad: Cualquier evento que comprometa la confidencialidad, integridad o disponibilidad de sistemas de información o datos, o que viole políticas de seguridad de la organización.

5.4.2 Fases de Respuesta a Incidentes

Fase 1: Preparación
  • Establecer equipo de respuesta a incidentes (CSIRT)
  • Desarrollar políticas y procedimientos
  • Implementar herramientas de monitoreo y detección
  • Realizar entrenamientos regulares
  • Mantener inventario de activos críticos
  • Establecer canales de comunicación seguros
Fase 2: Detección e Identificación
  • Monitorear sistemas continuamente
  • Analizar alertas de seguridad
  • Verificar y validar incidentes potenciales
  • Clasificar severidad del incidente
  • Documentar evidencias iniciales
  • Notificar al equipo de respuesta
Nivel de Severidad Descripción Tiempo de Respuesta Ejemplos
Crítico Impacto severo en operaciones Inmediato (< 1 hora) Ransomware, violación masiva de datos
Alto Impacto significativo 2-4 horas Compromiso de sistemas críticos
Medio Impacto moderado 8-24 horas Malware en sistemas no críticos
Bajo Impacto mínimo 1-3 días Intentos de phishing fallidos
Fase 3: Contención

Contención Inmediata:

  • Aislar sistemas comprometidos de la red
  • Preservar evidencias forenses
  • Implementar medidas temporales de protección
  • Evaluar alcance del compromiso

Contención a Largo Plazo:

  • Aplicar parches de seguridad
  • Cambiar credenciales comprometidas
  • Implementar controles adicionales
  • Preparar sistemas para recuperación
Fase 4: Erradicación y Recuperación

Erradicación:

  • Eliminar malware de todos los sistemas
  • Cerrar vulnerabilidades explotadas
  • Mejorar defensas existentes
  • Verificar eliminación completa de amenazas

Recuperación:

  • Restaurar sistemas desde backups limpios
  • Reconstruir sistemas comprometidos
  • Implementar monitoreo adicional
  • Validar funcionamiento normal
  • Restablecer operaciones gradualmente
Fase 5: Lecciones Aprendidas
  • Realizar revisión post-incidente
  • Documentar cronología completa
  • Identificar mejoras en procesos
  • Actualizar planes y procedimientos
  • Proporcionar entrenamiento adicional
  • Informar a stakeholders relevantes
Consideraciones Legales y Regulatorias:
  • Notificación a autoridades regulatorias (RGPD: 72 horas)
  • Notificación a individuos afectados
  • Preservación de evidencias para investigación
  • Cooperación con fuerzas del orden
  • Documentación para seguros y responsabilidad
  • Cumplimiento de obligaciones contractuales

5.4.3 Comunicación Durante Incidentes

Comunicación Interna:

  • Notificación inmediata al equipo de respuesta
  • Escalación a nivel ejecutivo según severidad
  • Actualizaciones regulares sobre progreso
  • Coordinación entre equipos técnicos

Comunicación Externa:

  • Notificaciones regulatorias obligatorias
  • Comunicación con clientes y socios
  • Coordinación con proveedores de servicios
  • Gestión de relaciones públicas y medios
