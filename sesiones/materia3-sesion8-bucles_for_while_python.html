Sesión 8: Bucles (for y while) en Python

1️ Introducción

Teoría:

  • Un bucle permite repetir un bloque de código varias veces.
  • Ventajas:
    • Ahorra código.
    • Automatiza tareas repetitivas.
    • Facilita el manejo de colecciones de datos.

2️ Bucle for

Teoría:

  • Recorre una secuencia (lista, tupla, string, rango, etc.).
  • Sintaxis: 
    for variable in secuencia:
    # código a ejecutar

Ejemplos:

# Recorrer un rango de números
# El valor ingresado en range() será la cantidad de posiciones que recorrerá
for i in range(5):
    print(i) # 0, 1, 2, 3, 4

for x in range(2, 6):
    print(x) # 2 3 4 5

# range(inicio, fin, paso) para generar secuencias numéricas.
for x in range(2, 30, 3):
    print(x) # 2 5 8 11 14 17 20 23 26 29

# Recorrer una lista
frutas = ["manzana", "banana", "pera"]
for fruta in frutas:
    print(fruta)

# Recorrer un string
for letra in "Python":
    print(letra)

ELSE en FOR:

Se puede usar else para que ejecute una acción luego de concretar un recorrido. 

for x in range(6):
    print(x)
else:
    print("Fin de bucle recorrido!")

BREAK en FOR:

Al usar break dentro de un bucle este interrumpirá toda acción (corta el bloque en ejecución). 

for x in range(6):
    if x == 4: break
    print(x)
else:
    print("Fin de bucle recorrido!") # 0 1 2 3

CONTINUE en FOR:

for i in range(10):
    if i % 2 == 0:
        continue
    print(i) # 1 3 5 7 9
# Cada valor por posición divisible por 2 que dé resto 0, es decir, los números pares cumplen la igualdad del if i % 2, es decir, 0 == 0, ingresando a continue, saltando el recorrido en acción y pasando a la siguiente iteración, por esto nunca se imprimen los valores pares.

Ejercicio:

  • Imprimir todos los números pares del 0 al 20.

3️ Bucle while

Teoría:

  • Ejecuta un bloque mientras la condición sea True.
  • Sintaxis: 
    while condicion:
    # código a ejecutar
  • Riesgo: bucles infinitos si la condición nunca cambia.

Ejemplos:

# Contador
contador = 0
while contador < 5:
    print(contador)
    contador += 1
# El final de cada recorrido incrementa en uno a contador, una vez llegue a 5 comparará 5 < 5 dando FALSE interrumpiendo el bucle.

# Adivinar contraseña
clave = "python123"
entrada = ""
while entrada != clave:
    entrada = input("Introduce la contraseña: ")
print("Acceso concedido")
# Hasta no ingresar la clave, de forma infinita solicitará la password.

Ejercicio:

  • Pedir un número al usuario hasta que ingrese uno mayor que 10.
  • Adivinar un número del 1 al 15. 2 input, uno para que tu compañero ingrese el número y otro dentro del bucle para que adivine el número.

4️ Control del flujo: break y continue

Teoría:

  • break: termina el bucle.
  • continue: salta a la siguiente iteración.

Ejemplos:

# BREAK en WHILE:
i = 1
while i < 6:
    print(i)
    if i == 3:
        break
    i += 1 # 1 2 3

# CONTINUE en WHILE
# Saltea el recorrido en acción y pasa al siguiente (en este ejemplo cuando i = 3 cumple la condición de 3 = 3 dando TRUE ingresando a continue, saltando el print() de i = 3)
i = 0
while i < 6:
    i += 1
    if i == 3:
        continue
    print(i) # 1 2 4 5 6

Ejercicio rápido:

  • Recorrer una lista y dejar de imprimir si aparece "stop".

5️ Ejercicios prácticos

Objetivo:

Aplicar for, while, break y continue.
  1. Tabla de multiplicar
    • Pedir un número y mostrar su tabla usando for.
  2. Cuenta regresiva
    • Desde un número ingresado hasta 0 usando while.
  3. Suma acumulada
    • Pedir números al usuario y sumarlos hasta que ingrese 0 (usar while y break).
  4. Filtrar positivos
    • Recorrer una lista y mostrar solo números positivos (continue para saltar los negativos).
  5. Juego de adivinanza
    • El programa genera un número aleatorio (1–20) y el usuario intenta adivinarlo usando while. Indicar si es mayor o menor.

6️ Desafío final

Problema:

  • Escribir un programa que:
    1. Pida un texto.
    2. Cuente cuántas vocales hay usando un for.
    3. Muestre el resultado.

7️ Cierre

Resumen:

  • for → recorrer secuencias.
  • while → repetir mientras la condición sea verdadera.
  • break y continue → controlan el flujo interno.
  • Elegir for cuando se conoce cuántas veces se repite, while cuando depende de una condición.
