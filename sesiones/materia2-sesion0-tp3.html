  • Debe incluir los nombres y apellidos de todos los integrantes.
  • Fecha de entrega: En cuanto puedan para su revisión.
  • Fecha defensa: los últimos dos miércoles de Noviembre. (Se evaluará lo realizado para comprobar que todo el equipo tiene los conocimientos básicos).

Temática del Proyecto

Deberán crear un sistema completo de base de datos. Pueden elegir UNA de las siguientes temáticas:

  • Opción 1: Sistema de Biblioteca.
  • Opción 2: Sistema de Tienda Online.
  • Opción 3: Sistema de Hospital.
  • Opción 4: Sistema de Escolaridad.
  • Opción 5: Temática libre.

Actividades por Tema

Creación de Base de Datos y Tablas

Objetivo: Evaluar el diseño completo de una base de datos con múltiples tablas relacionadas.

Requisitos:

  • Crear la base de datos con un nombre descriptivo (temático).
  • Crear un MÍNIMO de 4 tablas (evitar excederse en la cantidad de columnas y tablas).
  • Las tablas deben estar relacionadas entre sí.
  • Implementar los tres tipos de relaciones:
    • Relación 1:1 (uno a uno).
    • Relaciones 1:N (uno a muchos).
    • Relación N:M (muchos a muchos), (con tabla intermedia).

Cada tabla debe contener:

  • Columna id como PRIMARY KEY con AUTO_INCREMENT.
  • Cada tabla debe contener al menos 3 columnas (incluyendo “ID”).
  • Usar los tipos de datos: INT, VARCHAR, DOUBLE, DECIMAL, BOOLEAN, TEXT, DATE.
  • Aplicar NOT NULL donde corresponda.
  • Aplicar UNIQUE donde sea necesario.
  • Definir todas las FOREIGN KEY con:
    • ON DELETE CASCADE.
    • ON UPDATE CASCADE.

Incluir código para:

  • CREATE DATABASE IF NOT EXISTS.
  • USE.
  • CREATE TABLE IF NOT EXISTS.
  • DROP TABLE.
  • DROP DATABASE.

Ejemplo de estructura (Sistema de Biblioteca):

  • Tabla: bibliotecas (relación 1:1 con bibliotecarios).
  • Tabla: bibliotecarios.
  • Tabla: autores (relación 1:N con libros).
  • Tabla: libros.
  • Tabla: socios (relación 1:N con préstamos).
  • Tabla: préstamos (tabla intermedia para relación N:M entre socios y libros).

Inserción Masiva de Datos

Objetivo: Poblar todas las tablas con datos realistas y coherentes.

Requisitos:

  • Insertar MÍNIMO 3 registros en cada tabla principal.
  • Insertar MÍNIMO 6 registros en tablas intermedias.
  • Algunos registros deben tener valores NULL donde esté permitido.

Operaciones de Actualización y Eliminación

Objetivo: Demostrar el uso de UPDATE, DELETE y la integridad referencial.

Realizar:

  1. UPDATE
    • Actualizar usando WHERE con una condición simple.
    • Actualizar usando WHERE con AND/OR.
    • Actualizar con LIMIT.
  2. DELETE
    • Eliminar registros con WHERE.
    • Demostrar ON DELETE CASCADE eliminando un registro padre.
  3. Demostración de UPDATE CASCADE
    • Actualizar una PRIMARY KEY y mostrar cómo se actualizan las FOREIGN KEY automáticamente.
    • Documentar con comentarios el comportamiento observado.
  4. Uso de TRUNCATE
    • Vaciar una tabla y volver a llenarla.

Consultas con Condiciones

Objetivo: Aplicar filtros complejos y ordenamiento.

Crear consultas usando:

  1. WHERE con operadores:
    • AND, OR, NOT (3 consultas).
    • >=, <=, <>, =, >, < (3 consultas).
    • IS NULL, IS NOT NULL (2 consultas).
  2. ORDER BY:
    • ASC y DESC (2 consultas).
  3. LIMIT:
    • Seleccionar los primeros N registros.
    • Combinado con ORDER BY para Top N.
  4. DISTINCT: (1 consulta)
    • Usar DISTINCT.

Modificación de Estructura de Tablas

Objetivo: Demostrar el uso de ALTER TABLE.

Realizar:

  1. ADD COLUMN
    • Agregar una nueva columna a una tabla existente.
  2. MODIFY COLUMN
    • Cambiar el tipo de dato de una columna.
  3. CHANGE COLUMN
    • Renombrar una columna y cambiar su tipo.
  4. DROP COLUMN
    • Eliminar la columna creada para este ejercicio de una tabla.

  1. Planificar antes de programar: Hacer un diagrama o planificar las tablas y sus relaciones.
  2. Probar cada consulta antes de agregar la siguiente.

  4. Guardar el código frecuentemente.
  5. Consultar las sesiones anteriores como referencia (no se tienen que memorizar los códigos, sino saber que existen y para qué sirven).
  6. No dejar todo para el último día.
