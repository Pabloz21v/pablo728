Clase Teórico-Práctica: Operadores Lógicos, de Identidad y Pertenencia en Python

1. Operadores Lógicos

Los operadores lógicos ( and, or, not ) se utilizan para combinar o invertir valores booleanos ( True o False). También pueden trabajar con valores no booleanos, interpretando "truthy" (verdaderos) o "falsy" (falsos).

# and
  • Devuelve True si ambos operandos son verdaderos.
  • Si el primer operando es False, retorna el primer operando sin evaluar el segundo (cortocircuito).
print(5 > 3 and 2 < 4) # True (ambas condiciones son True)
print(0 and 10) # 0 (primer operando "falsy")

# or
  • Devuelve True si al menos un operando es verdadero. o
  • Si el primer operando es True, retorna el primer operando sin evaluar el segundo.
print(5 < 3 or 2 < 4) # True (segunda condición es True)
print("Hola" or 0) # "Hola" (primer operando "truthy")

# not
  • Invierte el valor booleano de un operando.
print(not True) # False
print(not 0) # True (0 es "falsy")

Ejercicios Prácticos

  • ¿Qué devuelve not (10 > 5 and 3 == 3)?
    Solución: False
  • Evalúa: "Python" or 0
    Solución: "Python"

2. Operadores de Identidad (is, is not)

Verifican si dos variables apuntan al mismo objeto en memoria (no si tienen el mismo valor).

1. is

  • True si ambos operandos son el mismo objeto.
a = [1, 2]
b = a
c = [1, 2]

print(a is b) # True (b referencia a la misma lista que a)
print(a is c) # False (listas diferentes en memoria)

2. is not

  • True si los operandos no son el mismo objeto.
x = 5
y = 5
print(x is not y) # False (enteros inmutables pueden compartir memoria)

Notas

  • Para tipos inmutables (como int, str), Python puede optimizar la memoria, haciendo que x is y sea True si tienen el mismo valor.
  • Para tipos mutables (como list, dict), is solo es True si son el mismo objeto.

Ejercicios Prácticos

  • ¿Qué retorna "Hola" is "Hola"?
    Solución: True (strings inmutables pueden compartir memoria)
  • Dado lista1 = [1, 2] y lista2 = lista1.copy(), ¿qué devuelve lista1 is lista2?
    Solución: False (son copias diferentes)

3. Operadores de Pertenencia (in, not in)

Verifican si un valor existe en una secuencia (lista, tupla, string, etc.).

1. in

  • True si el valor está presente en la secuencia.
numeros = [1, 2, 3]
print(2 in numeros) # True

texto = "Python"
print("th" in texto) # True

2. not in

  • o True si el valor no está en la secuencia.
vocales = {"a", "e", "i"}
print("u" not in vocales) # True

Ejercicios Prácticos

  • Dado dicc = {"a": 1, "b": 2}, ¿qué devuelve "a" in dicc?
    Solución: True (busca en las claves)

  • Evalúa: 10 not in range(5, 15)
    Solución: False (10 está en el rango 5-14)

Tarea

  • Operadores Lógicos: Evalúa not (False or 0) y explica el resultado.
  • Operadores de Identidad: Dado a = 256 y b = 256, ¿qué retorna a is b? ¿Y si a = 257 y b = 257?
  • Operadores de Pertenencia: Crea una lista frutas = ["manzana", "banana", "naranja"] y verifica si "pera" not in frutas.

Soluciones a la Tarea

# Operadores Lógicos
print(not (False or 0)) # True (False or 0 → 0, not 0 → True)

# Operadores de Identidad
a = 256
b = 256
print(a is b) # True (optimización de enteros pequeños en Python).

a = 257
b = 257
print(a is b) # False (no aplica optimización, objetos distintos).

# Operadores de Pertenencia
frutas = ["manzana", "banana", "naranja"]
print("pera" not in frutas) # True

Errores Comunes

Confundir is con ==:

  • is compara identidad (mismo objeto en memoria),
  • == compara igualdad de valores.

Usar in con tipos no iterables:

  • print(5 in 10) # Error: los enteros no son iterables.
