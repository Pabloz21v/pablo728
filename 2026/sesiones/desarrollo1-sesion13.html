Clase 13: Condicional Simple y Doble (if / else)

Marco Teórico

Hasta ahora nuestros programas ejecutaban las instrucciones siempre en el mismo orden, de arriba hacia abajo (estructura secuencial). Pero los problemas reales requieren tomar decisiones: hacer una cosa u otra según se cumpla o no una condición. Para eso usamos la estructura selectiva.

En Python, la estructura básica es el if. Si la condición es True, se ejecuta el bloque interno; si es False, se salta. El else define qué hacer cuando la condición no se cumple. La indentación (4 espacios o 1 Tab) es obligatoria en Python para indicar qué instrucciones pertenecen a cada bloque.

Operadores de Comparación

  • == → igual a
  • != → distinto de
  • > → mayor que
  • < → menor que
  • >= → mayor o igual que
  • <= → menor o igual que

Sintaxis

if condicion: # bloque que se ejecuta si la condición es True else: # bloque que se ejecuta si la condición es False

Ejemplos

Ejemplo 1 — if simple:

edad = int(input("¿Cuántos años tenés? ")) if edad >= 18: print("Eres mayor de edad.") print("Fin del programa.") # Esta línea siempre se ejecuta

Ejemplo 2 — if / else:

nota = float(input("Ingresá tu nota: ")) if nota >= 6: print("Aprobaste ✓") else: print("Desaprobaste ✗")

Ejemplo 3 — comparar strings:

clave = input("Ingresá la contraseña: ") if clave == "python2024": print("Acceso permitido.") else: print("Contraseña incorrecta.")

Ejemplo 4 — número positivo o negativo:

num = int(input("Ingresá un número: ")) if num >= 0: print("El número es positivo o cero.") else: print("El número es negativo.")

💡 Tip para el docente

Insistir en dos puntos clave:

  1. El == (comparación) es distinto del = (asignación). Es el error más común.
  2. La indentación es parte de la sintaxis en Python; un bloque mal indentado genera error. Proponer que los estudiantes deliberadamente rompan la indentación para ver qué mensaje arroja VS Code.

Actividad 13: Programas con if / else

Crea un archivo .py para cada uno de los siguientes ejercicios o agrupa varios en uno solo:

  1. Pedir un número al usuario. Si es mayor a 0, imprimir "Positivo", si no "Negativo o cero".
  2. Pedir la temperatura en °C. Si es mayor a 30, imprimir "Hace calor 🌞", si no "Temperatura agradable 🌤".
  3. Pedir la nota del alumno (0-10). Si es ≥ 6, mostrar "Aprobado", si no "Reprobado".
  4. Pedir el nombre del usuario. Si el nombre tiene más de 5 letras, mostrar "Nombre largo", si no "Nombre corto". Pista: usar len(nombre).
  5. Pedir dos números. Mostrar cuál es el mayor (o si son iguales).
  6. Programa de descuento: pedir el precio de un producto. Si el precio es mayor a $1000, aplicar 10% de descuento e imprimir el precio final; si no, imprimir el precio sin cambios.
  7. Pedir la contraseña al usuario. Si es "1234", mostrar "Bienvenido al sistema", si no "Acceso denegado".
  8. Pedir un número. Verificar si es par o impar usando el operador módulo % (par si num % 2 == 0).
  9. Pedir la cantidad de horas trabajadas. Si es mayor a 8, indicar que hay horas extras; si no, indicar jornada normal.
  10. Pedir un número del 1 al 10. Si el número está entre 1 y 5 (inclusive), mostrar "Mitad baja"; si está entre 6 y 10, mostrar "Mitad alta". Pista: usar AND para combinar condiciones.
