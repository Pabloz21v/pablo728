✅ Clase 13: Booleanos, Operadores de Comparación, Operadores Lógicos y Condicionales if

Aprende a dotar de inteligencia a tus programas para que tomen decisiones por sí mismos.

🔄 Repaso previo Hasta ahora, nuestros programas eran lineales: ejecutaban las instrucciones una por una, de arriba hacia abajo, sin excepción. Aprendimos a pedir datos con input(), descubrimos que siempre devuelve un texto (str) con la función type(), y vimos cómo hacer matemáticas con los operadores aritméticos básicos.   • Aprender a usar los operadores de comparación para evaluar datos.

⚖️ 1. Qué es un dato booleano (bool)

En el mundo real, muchas preguntas tienen solo dos respuestas posibles: ¿Está lloviendo? (Sí o No), ¿El usuario está registrado? (Verdadero o Falso). En Python, este tipo de información se maneja con los datos booleanos. Tienen solo dos valores posibles que siempre deben escribirse con la primera letra en mayúscula:

  • True (Verdadero)
  • False (Falso)
llueve = True
es_de_noche = False

print(type(llueve))  # Devuelve <class 'bool'>
#Observen que estas palabras True y False se ingresan sin comillas.
#Y al usar la funciín type() verificamos que se interpreta como booleano
#Notaremos en el VSC que toma otro color.

# ¡Ojo con las comillas!
falso_texto = "False" 
print(type(falso_texto))  # Devuelve <class 'str'> (porque tiene comillas)
#Si lo escribimos con comillas se interpreta como texto

🔍 2. Operadores de Comparación

Para que la computadora decida, primero tiene que comparar. Los operadores de comparación toman dos valores, los evalúan y devuelven un resultado booleano (True o False).

Observar como luego de comprar 2 datos, sean textos o números el resultado de esa comparación es un boleando, True o False según si dicha comparación es afirmativa o no.


⚠️ Peligro: = vs ==

Un solo igual = sirve para guardar (asignar) un valor en una variable.
Dos iguales == sirven para comparar si dos cosas son idénticas.

edad = 18 (Guarda 18 en edad)
edad == 18 (Pregunta: ¿Es la edad igual a 18?)

🧠 3. Operadores Lógicos

A veces una sola comparación no basta. ¿Qué pasa si quiero saber si un usuario tiene más de 18 años Y tiene licencia de conducir? Usamos los operadores lógicos:

Operador Descripción Ejemplo
and Devuelve True solo si AMBAS condiciones son verdaderas. (10 > 5) and (8 < 10) → True
or Devuelve True si AL MENOS UNA condición es verdadera. (10 > 50) or (8 < 10) → True
not Invierte el resultado. Cambia True a False y viceversa. not (5 == 5) → False

🔀 4. Estructuras Condicionales (if)

La instrucción if (que significa "si" condicional en inglés) evalúa una expresión. Si el resultado es True, ejecuta el bloque de código que tiene dentro. Si es False, lo salta. 

# Estructura básica de un if
edad = 20

if edad >= 18:
    print("Eres mayor de edad.")
    print("Puedes pasar.")

print("Este mensaje se imprime siempre.")

Cuando ejecutamos el código: en la línea 
if edad >= 18 
la maquina compara el valor guardado dentro de la variable edad (en este caso el numero 20) con el número 18.
Podemos afirmar que 20 es mayor que 18 por lo que es un TRUE. Al ser TRUE ejecuta las 2 líneas indentadas 
    Eres mayor de edad. 
    Puedes pasar. 
Si cambiamos el valor 20 por uno menor a 18, no ejecutara las líneas indentadas.

🛑 MUY IMPORTANTE: La Indentación (Sangría)

En otros lenguajes se usan llaves {} para decir qué código está dentro del if. En Python usamos espacios en blanco al principio de cada línea. A eso se le llama indentación, y no es un adorno visual: es obligatoria. Python la utiliza para saber exactamente qué instrucciones pertenecen al bloque del if y cuáles no.

Dicho de forma muy simple: cuando escribes un if, la línea termina con dos puntos : y, justo debajo, debes desplazar hacia la derecha las instrucciones que quieres ejecutar si la condición se cumple. Lo habitual y recomendable es usar 4 espacios.

  • Todo lo que tenga 4 espacios de margen debajo del if, pertenece al if.
  • Si una línea vuelve al margen normal, Python entiende que el bloque del if ha terminado.
  • Si olvidas la indentación, Python lanzará un error como IndentationError y el programa no se ejecutará.
  • NUNCA mezcles tabulaciones con espacios. Lo más seguro es configurar Visual Studio Code para trabajar siempre con espacios. 
# ❌ INCORRECTO (Dará un IndentationError)
if 5 > 2:
print("Es mayor")

# ✅ CORRECTO
if 5 > 2:
    print("Es mayor")  # Estos 4 espacios indican que esta línea pertenece al if.
temperatura = 32

if temperatura > 30:
    print("Hace mucho calor")
    print("Bebe agua")

print("Fin del programa")

Observar bien el ejemplo anterior: las dos líneas con 4 espacios se ejecutan solo si temperatura > 30 es verdadero. En cambio, print("Fin del programa") está pegado al margen izquierdo, por eso se ejecuta siempre, independientemente del resultado del if.

Recuerdar esta idea clave: una línea indentada está dentro del condicional; una línea sin indentación, está fuera. En Python, entender esto es imprescindible para trabajar con if, elif, else, bucles y funciones.

Añadiendo alternativas: else y elif

¿Y si la condición no se cumple? Para eso usamos else (sino). Y si queremos evaluar varias opciones en cadena, usamos elif (sino, si...). 

nota = 7

if nota >= 9:
    print("¡Sobresaliente!")
elif nota >= 6:
    print("Aprobado, bien hecho.")
else:
    print("Reprobado, necesitas estudiar más.")
  
#Observemos como dependiendo del número que guardemos en la variable nota se imprime determinado texto.
#Para el valor nota = 7 primero pasar por if nota es mayor o igual a 9, no entrega un FALSE, motivo por el que no ingresa a:  print("¡Sobresaliente!")
#Continua con nota es mayor o igual a 6, esta comparación da TRUE, por lo que pasa a imprimir: Aprobado, bien hecho.
#En este caso el código no ejecuta else ya que previamente se cumplió una condición. Dado el caso que ninguna condición previa se cumpla else se ejecutará.
#Pueden cambiar el valor guardado en la variable nota para probar los distintos resultados.

🔄 5. Convertir input() a números (str a int/float)

Recordemos la clase anterior: input() siempre devuelve texto. Esto significa que, aunque el usuario escriba 25 por teclado, Python lo recibe como "25". A simple vista parece un número, pero internamente sigue siendo una cadena de texto. 

numero = 25
print(type(numero))      #En consola veríamos: <class 'int'>
La máquina lo interprete como un numero entero (int-integer)

numero = "25"
print(type(numero))      #En consola veríamos: <class 'str'>
La máquina lo interprete como un texto (str-string-cadena de texto)

numero = input("Escribe un número: ")
print(type(numero))      #En consola veríamos: <class 'str'>
Lo ingresado por medio de un input la maquina lo interpreta como un texto, no importa si ingresamos un número.

numero = int(input("Escribe un número entero: "))
print(type(numero))      #En consola veríamos: <class 'int'>
Para que la maquina interprete lo ingresado por un input como numero entero debemos agregar int() tal y como ven en este ejemplo.

precio = float(input("Escribe un precio: "))
print(type(precio))      #En consola veríamos: <class 'float'>
Para que la maquina interprete lo ingresado por un input como numero decimal debemos agregar float() tal y como ven en este ejemplo.

Fíjate en la diferencia:

  • numero = 25 guarda un número entero real.
  • numero = "25" guarda texto, no un número para calcular.
  • numero = input(...) también guarda texto, aunque el usuario teclee cifras.
  • numero = int(input(...)) convierte ese texto en entero.
  • precio = float(input(...)) convierte ese texto en decimal.

Analiza ahora este problema: 

numero_string = input("Ingresa un número: ")  # Si escribo 25, se guarda como "25"
if numero_string > 10:  # ¡ERROR! Python no puede comparar texto con números.
    print("El número es grande")

Para solucionar esto, debemos usar funciones de conversión de tipos. Envolvemos el input() dentro de int() para enteros, o float() para decimales.

❌ Sin conversión (se guarda como texto) ✅ Con conversión (se guarda como número) 
edad = input("Cuál es tu edad?: ")
# edad vale "25" (str)
 
edad = int(input("Cuál es tu edad?: "))
# edad vale 25 (int)
 
precio = input("Cuanto sale el producto?: ")
# precio vale "15.5" (str)
 
precio = float(input("Cuanto sale el producto?: "))
# precio vale 15.5 (float)

Observar como usando int() para enteros, o float() para decimales
de la forma expresada precio = int(input(...)) y precio = float(input(...)) serán guardadas como números permitiendo que podamos realizar cálculos matemáticos con ellos

📌 Ejemplo 9: Invertir la lógica (Uso de not)

El operador not se usa mucho cuando queremos evaluar si algo "no" está sucediendo o cuando tenemos una variable booleana y queremos actuar sobre su estado negativo. 

hay_stock = False

# "not hay_stock" se lee como "si NO hay stock"
if not hay_stock:
    print("Lo sentimos, el producto está agotado.")
else:
    print("Añadiendo producto al carrito de compras.")
  
#Primero veamos que se guarda el booleano False en la variable hay_Stock indicando que nos quedamos sin productos.
#Luego vemos que invertimos su valor usando "not" ósea de False a TRUE.
#Ya que ahora es TRUE ingresa dentro de la indentación (primer condición) imprimiendo: o sentimos, el producto está agotado.

❌ Errores Frecuentes

  • Usar = en lugar de == en un if: if edad = 18: lanzará error. Recuerda usar == para comparar.
  • Olvidar los dos puntos (:) al final del if/else: Son obligatorios para indicarle a Python que empieza un bloque nuevo.
  • Mala indentación: Poner el print() en el mismo nivel que el if en lugar de dejarle 4 espacios al margen.
  • Olvidar convertir el input(): edad = input() y luego hacer if edad > 18: dará TypeError (compara texto contra número).
  • Comparar texto con número: Si edad = input("Edad: "), entonces edad es str. Compararlo con 18 sin convertirlo provocará error.
  • Poner "True" con comillas: activo = "True" crea un texto, no un booleano. activo = True es lo correcto.

📌 Resumen de la clase

  • Los booleanos (bool) indican True (Verdadero) o False (Falso).
  • Comparamos con ==, !=, >, <, >=, <=.
  • Podemos unir condiciones con los operadores lógicos and, or, not.
  • La estructura if - elif - else nos permite crear caminos y decisiones en el código.
  • La indentación es vital: 4 espacios a la derecha marcan qué líneas pertenecen a un condicional.
  • Siempre que operes con números introducidos por el usuario, debemos convertir el texto recibido envolviéndolo en la función int() o float().
