TP y Examen 1°: MySQL (13 a 22 de abril)

Instrucciones Generales

TP: En grupos de entre 2-4 integrantes deberán realizar las siguientes actividades. Se dedicarán 2 clases (13 y 15 de abril) para dicho trabajo (pueden avanzar en casa).

IA: Pueden apoyarse con el uso de IA, tener cuidado en que la IA les agregue contenido no enseñado en clase. A la vez, analizar el código ofrecido por la IA e interpretarlo para juzgar si es lo que desean y comprender qué les entregó.

Advertencia: Enviar por correo los script.sql (es decir, su código). Este puede ser interpretado como potencial virus, bloqueando toda posibilidad de descargarlo y recuperarlo. Guardarlo en un archivo WORD, block de notas o similar.

Entrega del TP: Por EMAIL (fecha límite: 18 de abril, 23:59 hs).

  • Debe incluir los nombres y apellidos de los integrantes.
  • 1 tabla si son 1 o 2 integrantes.
  • 2 tablas si son más de 2 integrantes.

Examen: 20 y 22 de abril.

  • Se les preguntará a los miembros del grupo sobre el código utilizado para corroborar su entendimiento.
  • Se les solicitará cambiar partes de código para verificar la comprensión del código usado.

Calificación: Tendrán una nota grupal (por el TP) y una individual (examen oral y modificación del código trabajado).

1. Creación y Gestión de Bases de Datos y Tablas

Objetivo: Evaluar el uso de CREATE DATABASE, DROP DATABASE, USE, CREATE TABLE IF NOT EXISTS, DROP TABLE, y tipos de datos.

  • Las tablas pueden ser de la temática que más les guste.
  1. TABLA: Dicha tabla debe contener como mínimo 6 columnas aplicando los tipos de datos aprendidos.
    • Primera columna llamada id (no olvidar asignarla como clave primaria y auto incremental).
    • VARCHAR
    • INT
    • DOUBLE
    • BOOLEAN
    • TEXT
    • Usar NOT NULL en alguna/s columna/s (que tenga sentido por qué no debe estar vacía dicha o dichas columnas).
  2. Aplicar IF NOT EXISTS al crear tablas o DDBB.
  3. Incluir código para eliminar tanto las tablas como las DDBB.
  4. No olvidar usar USE para posicionarse en la DDBB donde trabajen.

2. Inserción y Consulta de Datos

Objetivo: Evaluar el uso de INSERT INTO, SELECT * FROM, y LIMIT.

  1. Insertar 10 registros por tabla (es decir, 10 filas.
  2. Insertar 4 registros más por tabla repitiendo algunos datos de los 10 primeros registros y dejando algunos espacios vacíos (precaución con dejar valores vacíos en la columna/s que designaron con NOT NULL).
  3. Mostrar todos los registros.
  4. Mostrar 5 registros.
  5. Mostrar 10 registros.

3. Actualización y Eliminación de Datos

Objetivo: Evaluar el uso de UPDATE, SET, WHERE, DELETE, y TRUNCATE.

  1. Actualizar el valor de algún registro basado en la columna con tipo de dato VARCHAR.
  2. Actualizar el valor de algún registro basado en la columna con tipo de dato INT.
  3. Actualizar los valores de la tabla con tipo de dato DOUBLE mayor (>) o menor (<) a un valor que establezcan.
  4. Actualizar los valores de la tabla con tipo de dato INT mayor o igual a (>=) o menor o igual a (<=) un valor que exista.
  5. Actualizar uno o más datos de la tabla con tipo de dato VARCHAR().
  6. Actualizar uno o más registros rompiendo la seguridad.
  7. Eliminar una fila/registro basado en su id.
  8. Eliminar un rango de registros.
  9. Eliminar uno o más registros rompiendo la seguridad.
  10. Vaciar la tabla.
  11. Volver a insertar los registros a la tabla.

4. Consultas con Condiciones

Objetivo: Evaluar el uso de WHERE, AND, OR, NOT, >=, <=, <>, =, >, <, IS NULL, IS NOT NULL.

  1. Crear un filtro usando AND que involucre 3 columnas.
  2. Crear un filtro usando OR que involucre 3 columnas.
  3. Crear un filtro usando NOT que involucre 2 columnas.
  4. Crear uno o más filtros que apliquen >=, <=, <>.
  5. Crear uno o más filtros que apliquen =, >, <.
  6. Crear un filtro usando IS NULL y otro usando IS NOT NULL.
  7. Crear tres filtros combinando: >=, <=, <>, =, >, <, AND, OR, NOT, IS NULL, IS NOT NULL.

5. Ordenamiento y Valores Distintos

Objetivo: Evaluar el uso de ORDER BY (ASC/DESC) y DISTINCT, RENAME.

  1. Ordenar los registros de mayor a menor por la columna que contenga el tipo de dato VARCHAR.
  2. Ordenar los registros de menor a mayor por la columna que contenga el tipo de dato DOUBLE.
  3. Ordenar los registros (ASC o DESC) de la columna que deseen y que se vean solo 7 registros.
  4. Mostrar los registros basado en la columna con tipo de dato INT.
  5. Mostrar qué existe en las columnas del tipo de dato VARCHAR() y BOOLEAN.
  6. Cambiar el nombre de las dos tablas.
  7. Cambiar el nombre de las tablas a sus nombres originales.
