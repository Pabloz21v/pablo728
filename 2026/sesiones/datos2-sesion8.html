🛡️ CLASE 8: Recuperación, Reparación y Backup de Datos

Desde un borrado accidental hasta la clonación forense.

🎯 Objetivo de la clase

Comprender cómo funciona el borrado lógico y físico de archivos, recuperar datos eliminados, dominar estrategias de backup (regla 3-2-1), reparar cabeceras corruptas, y realizar clonado de discos. Al final, aplicarás herramientas reales como Recuva, TestDisk y clonado bit a bit.

📁 1. Recuperación de Datos I (Lógica)

🧠 ¿Qué pasa cuando borramos un archivo?

Papelera de reciclaje vs borrado permanente: Al enviar un archivo a la papelera, sólo se mueve a una carpeta especial. Al vaciar la papelera o usar Mayús + Supr, el sistema no destruye los datos; simplemente marca ese espacio como “disponible para sobrescribir”.

🔍 Tabla de asignación de archivos (FAT / MFT): Los sistemas de archivos (FAT32, NTFS, exFAT) usan una tabla maestra donde registran nombres, ubicaciones y atributos. Cuando se “borra”, se elimina la entrada, pero los datos persisten hasta que otro archivo los pise.

✨ Analogía didáctica: Es como un libro de biblioteca: quitas la ficha del catálogo, pero el libro sigue en el estante. Sólo cuando otro estudiante trae un libro nuevo y ocupa ese lugar, se pierde el anterior.

🛠️ Actividad 1: Recuperación forense con Recuva / TestDisk

Usa un pendrive de prueba (sin información confidencial).

  1. Preparación: Inserta un USB vacío o con datos de muestra. Copia algunas imágenes o documentos. Luego bórralos con Mayús+Supr.
  2. Descarga Recuva (gratuito) o TestDisk (portable). Escanea el USB.
  3. Analiza los archivos encontrados, intenta recuperar al menos 2 archivos.
  4. Reflexión: ¿Qué archivos aparecen con nombres originales? ¿Qué ocurre si grabas nueva información en el USB?

📌 Comando útil con TestDisk (modo consola): permite recuperar particiones perdidas. Para archivos individuales, usa PhotoRec (incluido en TestDisk).

# Ejemplo simbólico (explicación)
testdisk → seleccionar disco → [Analyse] → [Quick Search] → listar archivos borrados

💾 2. Prevención y Backups: Regla 3-2-1

📀 Tipos de respaldo

  • Completo (Full): Copia toda la información. Lento pero fácil de restaurar.
  • Incremental: Solo copia los cambios desde la última copia (completa o incremental). Requiere menos espacio.
  • Diferencial: Copia todos los cambios desde el último backup completo. Restauración más rápida que incremental.

⚡ Regla 3-2-1 (estándar profesional)

🔹 3 copias de tus datos.
🔹 2 tipos de medios diferentes (disco duro, nube, cinta).
🔹 1 copia fuera de las instalaciones (off-site).

🚨 Ejemplo concreto: Documentos de clase: 1) original en laptop, 2) copia en disco externo, 3) copia en Google Drive. ¡Protección total!

🤖 Actividad 2: Planificación y ejecución de rutina automatizada de backup

Crear script o tarea programada (Windows Backup o FreeFileSync).

  1. Diseña una política: ¿cada cuánto harás backups? ¿Completo semanal + incremental diario?
  2. Usa la herramienta “Historial de archivos” de Windows o programa Duplicati / FreeFileSync.
  3. Configura una carpeta de origen (ej: “Mis Documentos”) y destino (USB externo o carpeta en la nube).
  4. Realiza una simulación: agrega un archivo nuevo, ejecuta backup incremental y verifica que solo se respalde el cambio.

💡 Para casa: Implementa la regla 3-2-1 con tus fotos del celular.

🗂️ 3. Recuperación de Datos II: Particiones (MBR/GPT)

El disco se divide en particiones mediante esquemas: MBR (Master Boot Record) soporta hasta 2 TB y 4 particiones primarias; GPT (GUID Partition Table) permite más particiones y discos enormes, además incluye redundancia.

🔧 Si la tabla de particiones se daña (por error de formateo, virus o corte eléctrico), el sistema operativo no ve las unidades, pero los datos suelen seguir intactos.

Esquema Tamaño máx disco Nº particiones Seguridad
MBR 2 TB 4 primarias Un solo sector boot
GPT 9.4 ZB 128 (Windows) Tabla de particiones respaldo

⚙️ Actividad 3: Simulación de daño en partición y recuperación

Usando máquina virtual o USB secundario, nunca en disco principal

  1. Crea un pendrive con al menos 8 GB y crea dos particiones usando DiskPart o GParted.
  2. Daña la tabla de particiones MBR (por ejemplo, sobrescribe los primeros sectores con dd if=/dev/zero of=/dev/sdX bs=512 count=1 en Linux, o con herramienta de simulación).
  3. Utiliza TestDisk para analizar y reescribir la tabla MBR o GPT: escanea y recupera el acceso a las particiones.
  4. Vuelve a montar el disco y verifica que los archivos estén accesibles.
💡 Dato clave: TestDisk puede reparar sectores de arranque dañados y reconstruir la estructura. Ideal para discos que de repente aparecen como “RAW”.

🔨 4. Reparación de Archivos Dañados (Cabeceras corruptas)

Muchas extensiones (JPG, PDF, DOCX) tienen una cabecera que identifica el tipo de archivo. Si la cabecera se corrompe, el sistema no sabe cómo abrirlo. Herramientas específicas pueden reescribir la firma.

  • Word (.docx): WinRAR puede extraer el contenido XML; a veces renombrar a .zip y reparar archivos internos.
  • Imágenes JPG: Utilidades como JPEG Repair, Stellar Phoenix o PhotoRec (recupera incluso con cabecera dañada).
  • PDF: Adobe Reader ya intenta reparar, o herramientas como PDF Repair Toolbox.
🔎 Estructura típica: archivo JPG saludable empieza con "FF D8 FF" en hexadecimal. Si ves otros valores, se puede reparar manualmente con un editor hexadecimal.

🧪 Actividad 4: Reparación de documento Word corrupto intencionalmente

Aprende sobre la fragilidad de los encabezados.

  1. Crea un archivo .docx de prueba (una frase).
  2. Ábrelo con un editor hexadecimal (HxD) y modifica los primeros 4 bytes a valores aleatorios. Guarda y comprueba que Word no lo abre.
  3. Intenta repararlo: renombra a .zip, extrae el contenido y abre "document.xml". Reconstruye un nuevo .docx importando la estructura limpia.
  4. Alternativa más sencilla: usa la función “Abrir y reparar” de Microsoft Word o prueba herramientas online (con cuidado con datos sensibles).

✅ Conclusión: incluso si un archivo parece irrecuperable, muchas veces el contenido real sobrevive.

🖥️ 5. Utilitarios de Clonado: Imágenes de disco bit a bit

El clonado forense no copia archivos sueltos, sino que genera una imagen exacta del dispositivo (sectores, incluso los borrados). Usado en recuperación de datos y peritajes.

Herramientas clave: DD (Linux/ WSL), FTK Imager (gratuito, interfaz gráfica), Clonezilla para clonado de discos.

📀 Actividad 5: Creación de una imagen de disco de un USB

Preservar evidencia.

  1. Inserta una memoria USB pequeña (con datos de ejemplo).
  2. Usa FTK Imager Lite (Windows) o el comando dd si estás en Linux o Mac:
    sudo dd if=/dev/sdb of=backup_usb.img bs=4M status=progress
  3. Verifica que la imagen tenga el mismo tamaño que el dispositivo original.
  4. Monta la imagen como unidad virtual (con OSFMount) y comprueba que se pueden explorar los archivos sin necesidad del USB físico.
  5. ¿Por qué útil? Para trabajar sobre una copia y no dañar el original durante el análisis forense.
⚠️ Cuidado: al elegir la unidad origen ("if") no equivoques el destino, podrías sobrescribir tu disco principal.

📖 Material de lectura: Fundamentos técnicos

📘 Resumen extendido para estudiar en casa

  • Borrado lógico vs físico: El borrado lógico solo elimina punteros en el MFT. El borrado seguro (sobrescritura) sí destruye los datos. Herramientas como DBAN o cifrado previenen recuperación.
  • Estructura interna FAT/NTFS: FAT utiliza una tabla de asignación de clústeres; NTFS emplea Master File Table ($MFT) con registros de cada archivo.
  • Particiones MBR: reserva 512 bytes para código de arranque y tabla de particiones (4 entradas de 16 bytes). GPT es más moderno con cabecera primaria y secundaria al final del disco.
  • Cabeceras de archivos más comunes: PDF: "%PDF", PNG: "‰PNG", ZIP: "PK", DOCX (basado en ZIP).
  • Backup diferencial vs incremental: Diferencial acumula cambios desde el último completo → ocupa más pero restaura rápido; Incremental guarda cambios desde la última copia de cualquier tipo → ahorra espacio pero restaura más lento.
  • Clonado bit a bit: genera un hash (MD5/SHA1) para verificar integridad forense.

🔗 Recursos recomendados: TestDisk Documentation, Recuva, Clonezilla y guía oficial de FTK Imager.
