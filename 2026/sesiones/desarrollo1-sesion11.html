⌨️ Clase 11: Entrada y Salida (input y print)

Haz que tus programas conversen con el usuario

🎯 Repaso breve En la sesión anterior (Clase 10) aprendimos qué son las variables: cajas donde guardamos datos. Pero hasta ahora, los datos los escribíamos directamente en el código. ¿Qué pasaría si queremos que sea la persona que usa el programa quien introduzca esos datos? Para que un programa sea verdaderamente interactivo y dinámico, necesitamos pedir datos al usuario. Hoy aprenderemos a abrir esa vía de comunicación.

🎯 Objetivos de la clase

  • Comprender el concepto de Entrada (Input) y Salida (Output) en programación.
  • Aprender a utilizar la función input() para capturar información del usuario.
  • Combinar input() con print() para crear programas interactivos.
  • Identificar cómo el ordenador interpreta los datos introducidos por teclado.
  • Aplicar buenas prácticas para evitar errores comunes de sintaxis y lógica.

📚 Marco Teórico: Entrada y Salida

En el mundo de la informática, llamamos Entrada (Input) a la información que ingresa al ordenador (por ejemplo, cuando tecleamos nuestro nombre). Llamamos Salida (Output) a la información que el ordenador nos devuelve o muestra en pantalla (por ejemplo, un mensaje de bienvenida).

🔹 ¿Qué hace input()?

La función input() detiene la ejecución del programa y queda esperando a que el usuario escriba algo en el teclado y pulse la tecla Enter. Generalmente, dentro de los paréntesis escribimos un mensaje (texto entre comillas) para indicarle al usuario qué debe escribir.

🔹 ¿Qué hace print()?

Como ya vimos, print() se encarga de la Salida. Imprime en la consola lo que le indiquemos: texto puro, variables, o combinaciones.

🔹 ¿Cómo guardamos lo introducido por el usuario?

Si solo usamos input(), el usuario escribirá algo, pero el programa lo olvidará inmediatamente. Para poder usar esa información más tarde, debemos guardarla en una variable usando el signo igual (=).

Comparativa: Código estático vs. Código interactivo

📦 Asignación directa (Clase 10) 💬 Asignación interactiva (Clase 11) 
nombre = "Ana"
apellido = "Robles"
edad = "76"
mascota = "Sultán"
El programador decide que el nombre siempre será "Ana", "Robles", etc. 
nombre = input("¿Cómo te llamas? ")
apellido = input("¿Cuál es tu apellido? ")
edad = input("¿Cuál es tu edad? ")
mascota = input("¿Cómo se llama tu mascota? ")
El programa pregunta y se adapta a lo que escriba el usuario.

💡 Ejemplos

📌 Ejemplo 1: Pedir nombre y saludar

# Pedimos el nombre al usuario y lo guardamos en la variable 'nombre'
nombre = input("¿Cómo te llamas? ")

# Saludamos utilizando la variable
print("Hola,", nombre)

Interpretación: El ordenador muestra el mensaje "¿Cómo te llamas? " y pausa el programa. Si el usuario escribe "Carlos" y pulsa Enter, la palabra "Carlos" se guarda en la variable nombre. Luego print() imprime "Hola," y el contenido de la variable. En pantalla veremos: Hola, Carlos.

📌 Ejemplo 2: Pedir ciudad y mostrar mensaje con f-strings

ciudad = input("¿En qué ciudad vivís? ")
print(f"¡Qué bonita debe ser {ciudad} en esta época del año!")

Interpretación: Usamos f-strings (cadenas formateadas). Al colocar la letra f antes de las comillas, podemos incrustar la variable ciudad dentro de llaves {}.

Linea 1: Primero solicita introducir una ciudad, la ingresamos, precionamos enter.

Linea 2: Imprimirá el mensaje con la ciudad que introducimos.

📌 Ejemplo 3: Pedir nombre y apellido por separado

nombre = input("Introduce tu nombre: ")
apellido = input("Introduce tu apellido: ")
print(f"Bienvenido/a {apellido}, {nombre}, te deseo un muy buen día")

Interpretación: Podemos usar tantos input() como necesitemos. Cada uno debe guardarse en una variable diferente. Al final, en el print() , podemos alterar el orden de las variables sin problema. Si el usuario escribe "María" y luego "García", el programa imprimirá: Bienvenido/a García, María te deseo un muy buen dia.

📌 Ejemplo 4: Simular un pequeño formulario

print("--- FORMULARIO DE REGISTRO ---")
usuario_nombre = input("Nombre: ")
usuario_edad = input("Edad: ")
usuario_correo = input("Correo electrónico: ")

print("\n--- DATOS GUARDADOS ---")
print(f"Usuario: {usuario_nombre}")
print(f"Edad: {usuario_edad} años")
print(f"Email: {usuario_correo}")

Interpretación:Este bloque simula un registro real. Nota cómo usamos print("\n...") . El símbolo \n crea un salto de línea (como pulsar Enter al escribir un documento) para que el resultado en consola quede visualmente más ordenado y separado.

📌 Ejemplo 5: Pausar el programa

input("Pulsa la tecla Enter para salir...")
print("Este programa ha terminado su tarea.")

Interpretación: Acá vemos un uso especial de input() sin guardarlo en una variable. No nos importa lo que el usuario escriba; solo queremos aprovechar la capacidad de input() para pausar el programa hasta que se pulse Enter. Esto es muy útil al crear programas ejecutables para que la ventana no se cierre de golpe al terminar.

⚠️ Atención: input() SIEMPRE devuelve texto

Este es uno de los conceptos más importantes que deben asimilar. La función input() está programada para capturar las pulsaciones del teclado como caracteres de texto (string - str ), sin importar si el usuario teclea letras o números. 

edad = input("¿Cuántos años tenés? ")
print(type(edad))   # <class 'str'>

Si el usuario escribe 25, visualmente parece un número, pero Python lo guarda como "25" (con comillas internas, es decir, texto). Si probas el código anterior, la consola mostrará (string).
¿Por qué es importante? Porque si intentamos hacer operaciones matemáticas con un texto, el programa fallará o hará cosas inesperadas (como sumar "25" + "1" y dar como resultado "251" en lugar de 26). En próximas clases aprenderemos a convertir esos textos a números enteros (int) o decimales (float).

❌ Errores Frecuentes al usar input()

  • Olvidar las comillas en el mensaje: ❌ nombre = input(Escribe tu nombre: ) → SyntaxError. ✅ nombre = input("Escribe tu nombre: ")
  • Olvidar dejar un espacio al final del mensaje: ❌ input("Nombre:") → el usuario escribirá pegado. ✅ input("Nombre: ")
  • Poner la variable entre comillas en el print: ❌ print("Hola, 'nombre'") → imprime literal "Hola, 'nombre'". ✅ print("Hola,", nombre)
  • Confundir input() con print(): ❌ input("El resultado es", variable) → usamos input() solo para pedir. ✅ para mostrar usamos print().
  • No asignar el input a ninguna variable: ❌ input("Dime tu color:") → la respuesta se pierde. ✅ color = input("Dime tu color: ")

📘 Ejercicios básicos (usamos input() y print())

  1. El saludo formal: Pide al usuario su nombre y luego salúdalo imprimiendo "Saludos cordiales, [nombre]".
  2. La mascota: Pregunta "¿Cuál es el nombre de tu mascota y/o mascotas?" y muestra un mensaje bonito que incluya esos nombres.
  3. Apellidos primero: Pide nombre y apellido en dos input() distintos y muestra: "Registrado en el sistema: [Apellido], [Nombre]".
  4. Geografía personal: Pide la ciudad donde vive y muestra: "Vives en la maravilla, genial, grandiosa, hermosa ciudad de [ciudad]".
  5. El eco culinario: Pregunta "¿Cuál es tu comida favorita?" y responde: "¡Mmm! A mí también me encanta comer [comida]".

⚙️ Ejercicios intermedios

  1. El orden inverso: Pide tres palabras diferentes con tres variables (palabra1, palabra2, palabra3) y luego imprímelas en una sola línea pero en orden inverso (tercera, segunda, primera).
  2. Completar la frase: Muestra "Por favor, completa la frase: Mi deporte favorito es..." Captura la respuesta y luego imprime la frase completa.
  3. Elección de color: Pide un color, imprime "Tu color elegido es [color]" y en la siguiente línea imprime el tipo de dato usando type() (observarás que es str).
  4. Pausa obligatoria: Crea un programa que imprima tres líneas de texto (una historia corta). Al final, usa un input() sin guardarlo que diga "Presione Enter para salir del cuento...".
  5. El entrevistador: Pide a un compañero su banda/cantante favorito y su canción preferida, luego muestra usando f-strings: "Acabas de sintonizar [canción] del gran artista [artista]".
  6. Repetición redundante: Pide una palabra cualquiera y luego usa un solo print() para imprimir esa misma variable cinco veces separadas por comas.
  7. El detective de tipos: Pide un número por teclado (ej. año de nacimiento). Imprímelo y en la línea siguiente imprime type() de esa variable para demostrar que Python lo considera texto. Añade un comentario explicando por qué.

📌 Resumen de la clase

  • input() es la herramienta fundamental para recibir datos del usuario (Entrada).
  • Todo programa interactivo sigue un ciclo: preguntar con input(), guardar en variable, procesar y mostrar con print() (Salida).
  • Fundamental: Todo lo que entra a través de input() es considerado texto (str) por el ordenador, incluso si tecleamos números.
  • Podemos combinar textos fijos y variables usando comas en print() o las más modernas f-strings.
  • Si usamos input() sin asignarlo a una variable, actúa simplemente como una pausa en la ejecución.
✨ Buenas prácticas

Siempre incluye un mensaje claro dentro de input() para guiar al usuario. Usa f-strings para mayor legibilidad. Recuerda convertir tipos cuando necesites números (lo veremos pronto).
