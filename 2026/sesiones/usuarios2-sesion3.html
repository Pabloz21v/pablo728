Sesión 3: Filtrado y Borrado en Bases de Datos

📍 Ver Bases de Datos, Tablas y Posicionarse

Teoría

Comandos básicos:

  • SHOW DATABASES; → Muestra todas las bases de datos.
  • SHOW TABLES; → Muestra las tablas de la base de datos activa.
  • USE nombreBD; → Selecciona una base de datos.

Práctica

  1. Escribir y ejecutar: SHOW DATABASES;
  2. Escribir y ejecutar: SHOW TABLES;
  3. Escribir y ejecutar: USE tu_database_para_continuar;

📍 Crear Tablas y Tipos de Datos

Teoría

Para crear una tabla:

CREATE TABLE alumnos ( id INT AUTO_INCREMENT PRIMARY KEY, nombre VARCHAR(50), edad INT, promedio DOUBLE );

AUTO_INCREMENT: Genera automáticamente valores consecutivos (suma 1 al número anterior).

En desarrollo 2 ustedes crean un id_nombre_tabla los valores de los ID (números) no deben repetirse dentro de la tabla. Ejemplo el DNI, son únicos, o las patentes de los autos. Al ser únicos permiten identificar algo de forma precisa.

Bueno para asegurar que cada vez que ingresen un nuevo alumno (para este ejemplo) al usar INSERT INTO alumnos (nombre,edad,promedio) no deben escribir la primer columna (ID) que crearon con CREATE TABLE alumnos, ya que esa columna aumentará de a 1 (+1) cada vez que ingresen un nuevo alumno, asegurando así que cada ID sea único.

DOUBLE: Tipo de dato para números con decimales.

En clases previas para indicar que se debe escribir un número en una columna, ejemplo: la columna EDAD, usaron INT. Bueno para indicar que el número puede llevar coma se usa DOUBLE. Por ejemplo, para un promedio, la cuenta bancaria o la plata en “mercado pago”, o sus calificaciones 9,5.

Práctica

  1. Crear la tabla alumnos.
  2. Insertar datos: INSERT INTO alumnos (nombre, edad, promedio) VALUES ('Juan', 20, 8.5), ('Ana', 22, 9.3), ('Pedro', 19, 7.2);
  3. Verificar con: SELECT * FROM alumnos;

📍 Consultas y Filtros

Teoría

Usar operadores permite filtrar datos de la tabla dependiendo de las condiciones que establezcan.

SELECT ... FROM ... WHERE

SELECT nombre, promedio FROM alumnos WHERE promedio > 6;

Interpretación: Seleccionar en las columnas nombre y promedio de la tabla alumnos los registros con un valor en promedio mayores a 6.

  • AND → Ambas condiciones deben cumplirse.
  • OR → Al menos una condición debe cumplirse.
  • NOT → Niega una condición.

Práctica

Crear varios alumnos e insertar los códigos para observar su comportamiento

Recuerden que han creado otras tablas llamadas alumnos, estas tablas viejas serán distintas a la sugerida en esta clase, la pueden dropear, o crear una nueva con otro nombre.

También pueden renombrar las viejas tablas usando la siguiente sentencia:

RENAME (renombrar tabla):

RENAME TABLE alumnos TO estudiantes;
  1. Seleccionar alumnos mayores de 20 años y con promedio > 8: SELECT * FROM alumnos WHERE edad > 20 AND promedio > 8;
  2. Seleccionar alumnos con promedio > 9 o edad < 20: SELECT * FROM alumnos WHERE promedio > 9 OR edad < 20;
  3. Seleccionar alumnos que no tengan promedio mayor a 8: SELECT * FROM alumnos WHERE NOT promedio > 8;

📍 Operaciones con Tablas y Registros

Teoría + Ejemplos

  • DELETE (borra registros): DELETE FROM alumnos WHERE id = 2;

    Borra la fila dentro de la tabla que cumpla lo indicado. En este ejemplo si encuentra en la columna nombre de la tabla alumnos con el valor = 'Pedro'; Borrará esa fila.

  • TRUNCATE (vacía toda la tabla, pero mantiene estructura): TRUNCATE TABLE alumnos;

    En vez de DELETE que borra solo la fila indicada, TRUNCATE borra todos los datos dentro de la tabla, dejándola vacía.

  • DROP TABLE IF EXISTS (elimina la tabla si existe): DROP TABLE IF EXISTS alumnos;

    Ya vieron como usar DROP, nada más que acá se agrega la sentencia “IF EXISTS”, es decir, si existe.

Práctica

  1. Insertar algunos registros, al menos 15 alumnos.
  2. Borrar un alumno específico.
  3. Borrar un rango de alumnos.
  4. Truncar la tabla.
  5. Insertar de nuevo datos y renombrar la tabla a estudiantes.
  6. Dropear la tabla definitivamente.
