Clase 10: 📦 Variables y Tipos de Datos

🔹 1. ¿Qué es una variable?

Una variable es un espacio en la memoria del ordenador que almacena un dato. Podes imaginarla como una caja con una etiqueta (nombre) que guarda un valor (número, texto, etc.). Ese valor puede cambiar durante la ejecución del programa.

Es recomendable nombrar las variables de forma que les sirva de guía sobre que puede estar conteniendo

📌 Ejemplos de asignación: 
edad = 25               #número entero
nombre = "Ana"          #texto (string)
altura = 1.65           #decimal
es_estudiante = True    #booleano

🏷️ 2. Cómo se puede (y no se debe) nombrar una variable

✅ Formas correctas (buenas prácticas)

  • Usar minúsculas y separar con guion bajo: precio_total, nombre_usuario
  • Comenzar con letra o guion bajo: _contador, valor1
  • Solo letras, números y guion bajo (sin espacios ni símbolos especiales ?, -, *) 
mi_variable = 10      #correcto
variable_2 = "Hola"   #correcto
_privado = True       #convención interna

❌ Formas incorrectas

  • No empezar con número: 2edad = 30 ❌
  • No usar espacios: mi variable = 5 ❌
  • No usar palabras reservadas: (propias del lenguaje): if, for, while, print, True, False if = 7, print = 3 ❌
  • Evitar caracteres especiales: mi-variable, mi$variable ❌ 
2edad = 30       #SyntaxError
mi variable = 5  #SyntaxError
if = 7           #SyntaxError

🖨️ 3. Incorporar variables en print()

Dos métodos principales para mostrar el contenido de variables junto con texto:

  • Método 1 (comas): print("Nombre:", nombre, "Edad:", edad) → añade espacios automáticos.
  • Método 2 (f-strings, recomendado): print(f"Nombre: {nombre}, Edad: {edad}") → más legible y potente. 
nombre = "Carlos"
edad = 20

print("El nombre es", nombre, "y tiene", edad, "años")

#Recordemos: lo que esta dentro de las comillas se interpreta como texto, la coma fuera de las comillas separa las variables de los textos.

print(f"El nombre es {nombre} y tiene {edad} años")

#Usando f"" dentro de print() podemos usar las variables al rodearlas con llaves {} y rodear todo por una apertura y cierre de comillas
Copiar el codigo y ejecutarlo. Podrán observar que el resultado es el mismo. Solo cambia la forma de escribir el codigo

📢 4. Imprimir variables directamente

También podemos pasar solo la variable a print():

precio = 99.90
print(precio)   # muestra 99.9

Si tenemos varias, usamos comas o f-strings como vimos.

💬 5. Comentarios en Python

Los comentarios son ignorados por Python. Sirven para explicar el código, deshabilitar líneas temporalmente o documentar. 

# Esto es un comentario de una línea
edad = 25  # comentario al final

"""
Comentario multilínea:
puedes explicar la lógica completa
o documentar una función.
"""

✨ Buenas prácticas Comenta el por qué no el qué. Útil para algoritmos complejos o tareas pendientes (# TODO:).

📊 6. Tipos de datos básicos y la función type()

Tipo Qué representa Ejemplo Uso típico
int Números enteros 10, -3, 0 Edades, cantidades, índices
float Números decimales 3.14, -0.5 Alturas, precios, promedios
str Texto (string) "Hola", 'Python' Nombres, mensajes, direcciones
bool Valores lógicos True, False Condiciones, banderas

Cómo guardarlos en variables

entero = 42
flotante = 3.1416
texto = "Python es genial"
booleano = True

🔎 Función type()

Nos muestra el tipo de dato de una variable o valor. Muy útil para depurar. 

edad = 25
print(type(edad))        # <class 'int'>

precio = 19.99
print(type(precio))      # <class 'float'>

nombre = "Luis"
print(type(nombre))      # <class 'str'>

es_mayor = False
print(type(es_mayor))    # <class 'bool'>

También puedes pasar directamente el valor:

print(type(3.14))        # <class 'float'class>

📌 Resumen de la clase

  • Variable: caja etiquetada que almacena un dato.
  • Nombres: solo letras, números, guion bajo; no empezar con número; sin espacios.
  • Print con variables: comas o f-strings (estos últimos más modernos).
  • Comentarios: # o triple comillas; ayudan a entender y depurar.
  • Tipos básicos: int, float, str, bool → usa type() para identificarlos.
  • Práctica: Escribir cuentos literales con variables, aplica buenas prácticas.
