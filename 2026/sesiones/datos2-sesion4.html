Clase 4: Integridad de Datos y Hashes

¿Llegó bien el mensaje? — El concepto de Integridad

❓ "Cuando descargas un juego de 50 GB o envías un contrato importante por email, ¿cómo sabes que el archivo no fue alterado durante el proceso?"

💥 Corrupción accidental

Errores durante la transmisión por fallas de red, problemas en el disco duro o interrupciones en la descarga.

🦠 Manipulación maliciosa

Un atacante (Man-in-the-Middle) intercepta el archivo y lo modifica para inyectar malware o alterar datos.

🔑 Definición de Integridad

La integridad de los datos garantiza que la información:

  • No fue modificada
  • No fue dañada
  • Se mantiene exactamente igual desde su origen

La "Huella Digital" del archivo: HASH

Un hash es el resultado de aplicar una función matemática a un archivo. Convierte cualquier entrada (imagen, video, PDF) en una cadena ÚNICA de caracteres de longitud fija.

📄 archivo.pdf ➜ ⚙ Función SHA-256 ➜ A1B2C3...F9

  • 🔁 Determinista: Siempre genera exactamente el mismo resultado para el mismo archivo de entrada.
  • 💥 Efecto Avalancha: Cambiar 1 solo bit en el archivo original cambia completamente el hash resultante.
  • 🚫 Irreversible: Es una función de un solo sentido. No se puede "des-hashear" para recuperar el archivo original.

Comparativa de Algoritmos Comunes

Algoritmo Longitud Seguridad Uso actual
MD5 32 caracteres (hex) ❌ Obsoleto Verificación rápida no crítica (vulnerable a colisiones)
SHA-1 40 caracteres (hex) ⚠ Débil En desuso por seguridad
SHA-256 64 caracteres (hex) ✅ Seguro Bancos, blockchain, firmas digitales, SSL

Ejemplo Real (Efecto Avalancha):

Texto: "Hola mundo" SHA-256:

b94d27b9934d3e08a52e52d7da7dabfac484efe04294e576e4d...

Texto: "Hola mund0" (solo cambia la última letra) SHA-256:

4a8a08f09d37b73795649038408b5f33...

¡Son completamente diferentes!

CHECKSUM — El Primer Nivel de Verificación

📌 Definición

Un checksum (suma de verificación) es un valor numérico calculado a partir de los datos de un archivo para detectar errores, muy útil para detectar perdida de paquetes (información que se pierde durante la transmisión, la cual llega incompleta al receptor). Es una técnica más antigua y simple que un hash criptográfico.

⚙ Cómo funciona

  1. Se suman los valores de los datos (bytes).
  2. Se obtiene un número final.
  3. Se envía junto al archivo.
  4. El receptor repite la suma y compara.

🔍 Ejemplo Conceptual

A = 65, B = 66, C = 67

Suma Original: 198

Error en transmisión (C cambia por D):

A = 65, B = 66, D = 68

Nueva Suma: 199

❌ ¡Error detectado!

⚠ Limitaciones del Checksum:

  • No es seguro contra ataques intencionados (fácil de falsificar).
  • No detecta modificaciones complejas (ej: cambiar el orden de los bytes podría dar la misma suma).
  • Su uso principal es detectar errores accidentales (ruido en la red, fallos de disco).

⚖ Tabla Comparativa: Hash vs Checksum

Característica Checksum (ej: CRC32) Hash Criptográfico (ej: SHA-256)
Complejidad Baja (Suma simple/XOR) Alta (Matemática compleja)
Seguridad ❌ Baja (Fácil de engañar) ✅ Alta (Casi imposible de engañar)
Resistencia a ataques No Sí
Uso principal Errores accidentales Seguridad e Integridad

"El checksum fue la base histórica → los hashes son su evolución segura."

¿Cómo se valida? — Proceso Completo

"El checksum fue la base histórica → los hashes son su evolución segura."

📤 EMISOR:

Archivo → Calcular SHA-256 → Hash Original

⬇ Se envían por internet (Archivo + Hash) ⬇

📥 RECEPTOR:

Descargar Archivo → Recalcular SHA-256 → Comparar

⚠ Puntos Críticos

  • Se debe usar el MISMO algoritmo en ambos lados.
  • El archivo debe estar 100% completo antes de calcular.
  • La comparación debe ser EXACTA (byte a byte).

🛠 Herramientas en Windows (Práctica)

  1. reamos una carpeta, dentro de la misma creamos un archivo .docx (Word) este archivo debe contener algún texto.
  2. Abrimos el CMD, esto se hace apretando en el teclado el logo de Windows y escribimos CMD
  3. Escribimos CD acompañado de un espacio “CD “
  4. Procedemos a desplazar la carpeta creada a la consola del CMD
  5. Si lo hicieron bien aparece mas texto.
  6. Apretamos ENTER, esto no ubicará dentro de la carpeta y detectará el fichero /archivo Word dentro.
  7. Ahora pegamos el código ofrecido. Respetar el nombre que le pusieron al archivo Word
  8. Adaptar el código ofrecido con el nombre que crearon y o tipo de extensión del arvhico creado.
  9. Lo pegamos en la consola y apretamos ENTER.
  10. Si todo salió bien les ofrecerá la información solicitada.

Símbolo del Sistema (CMD) - CertUtil

certutil -hashfile documento.docx MD5
certutil -hashfile documento.docx SHA256

PowerShell

Get-FileHash documento.docx -Algorithm SHA256

El Efecto Avalancha

Concepto: En los algoritmos de hash, cambiar una simple coma en un libro de 1000 páginas cambia el hash resultante por completo.

Preguntas:

  1. ¿Por qué esta característica es buena para la seguridad informática?
  2. ¿Qué pasaría si un cambio pequeño generara un hash muy similar al original?

🧠 Resumen de la Clase

Integridad

Garantía de que los datos no fueron alterados, dañados o modificados desde su origen hasta su destino.

Hash

Huella digital matemática única. Es determinista, irreversible y tiene efecto avalancha. SHA-256 es el estándar actual.

Checksum

Método simple y rápido para detectar errores accidentales. No sirve para seguridad contra ataques.

Herramientas

Windows: CertUtil / PowerShell. Python: Librería hashlib (seguridad) y zlib (checksums).

"Un solo bit diferente = Hash completamente distinto = Archivo comprometido"
