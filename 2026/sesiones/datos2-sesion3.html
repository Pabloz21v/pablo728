Clase 3: Formatos y Compatibilidad

🎯 Objetivos de Aprendizaje:

  • Distinguir entre formatos propietarios (cerrados) y abiertos.
  • Comprender la importancia de la interoperabilidad entre sistemas.
  • Conocer y aplicar formatos estándar de intercambio: CSV, TXT, XML, JSON.
  • Identificar los problemas más comunes de compatibilidad y sus causas.
  • Realizar conversiones manuales y con software de texto plano a planilla de cálculo.

📖 La Torre de Babel Digital

Imagina que le mandás una carta escrita en español a alguien que solo lee japonés. No hay comunicación posible, aunque las palabras estén físicamente escritas. En informática ocurre exactamente lo mismo: si un sistema guarda información en un formato que otro sistema no entiende, los datos existen pero son inaccesibles o se interpretan de forma incorrecta.

Este problema es más frecuente de lo que parece. Una empresa puede exportar datos desde su sistema de gestión en un formato específico y, al intentar importarlos en otro programa, encontrar que las fechas aparecen al revés, los nombres tienen símbolos raros o las columnas no coinciden. Ese caos tiene nombre: incompatibilidad de formatos.

En esta clase vamos a entender por qué sucede esto, qué formatos universales existen para evitarlo y cómo convertir datos correctamente sin perder información en el intento.

📚 Tipos de Formatos de Archivo:

Formatos Propietarios (Cerrados):

Son diseñados y controlados por una empresa específica. Solo el software oficial de esa empresa garantiza abrirlos y editarlos perfectamente.

Ejemplos: .PSD (Adobe Photoshop), .AI (Adobe Illustrator), .NUMBERS (Apple), .CDR (CorelDraw).

⚠ Riesgo: Si la empresa cierra, cambia el formato o deja de actualizar el programa, los archivos quedan "huérfanos" y pueden volverse ilegibles con el tiempo.

🔎 Caso Real — El Problema de WordPerfect:

En los años 90, WordPerfect era el procesador de texto más popular del mundo. Millones de documentos se guardaron en su formato propietario .WPD. Cuando Microsoft Word tomó el mercado, muchos de esos documentos quedaron inaccesibles porque ningún programa moderno los abría correctamente. Las empresas que habían guardado contratos y documentos legales en ese formato tuvieron que invertir tiempo y dinero en recuperarlos. La lección: los formatos abiertos protegen tu información en el futuro.

Formatos Abiertos (Libres):

Sus especificaciones técnicas son públicas. Cualquier desarrollador puede crear software para leerlos o escribirlos.

Ejemplos: .TXT, .CSV, .XML, .JSON, .PNG, .PDF/A, .ODT, .SVG.

✅ Ventaja: Son ideales para preservar información a largo plazo e intercambiar datos entre sistemas distintos.

📖 Los Cuatro Formatos Universales de Intercambio

📄 A. TXT — Texto Plano

Es el formato más básico y universal de todos. No tiene formato visual de ningún tipo: sin negritas, sin colores, sin tablas. Solo caracteres de texto. Cualquier sistema, en cualquier idioma y en cualquier época, puede abrir un archivo TXT.

Se usa para archivos de configuración de sistemas, logs (registros de eventos), scripts y cualquier situación donde la simplicidad sea prioritaria.

📊 B. CSV — Comma Separated Values (Valores Separados por Comas)

Es el formato de intercambio de datos tabulares (tablas) más usado del mundo. Ideal para mover datos entre Excel, Google Sheets, bases de datos y sistemas web. Cada línea es una fila y los valores de cada columna se separan por un delimitador (coma o punto y coma).

La primera línea generalmente contiene los encabezados de las columnas.

🔑 Reglas importantes del CSV:

  • Si un campo contiene el carácter delimitador, debe encerrarse entre comillas dobles. Ej: "Rodríguez, Ana" (el apellido con coma adentro va entre comillas).
  • Si un campo contiene comillas dobles, se duplican para "escaparlas". Ej: "Dijo ""hola""" representa el texto: Dijo "hola".
  • Las celdas vacías se representan con dos delimitadores juntos. Ej: Ana,,45123456 (sin nombre del medio).

🏷 C. XML — eXtensible Markup Language

XML es un lenguaje de marcado diseñado para almacenar y transportar datos de forma estructurada y autodescriptiva. Usa etiquetas (similar a HTML) que el propio usuario define según sus necesidades.

A diferencia del CSV, XML puede representar estructuras de datos complejas y jerárquicas (datos dentro de datos). Es muy usado en sistemas empresariales, servicios web, archivos de configuración y facturación electrónica (la AFIP en Argentina usa XML para las facturas electrónicas).

🌐 D. JSON — JavaScript Object Notation

JSON es el formato moderno y preferido para intercambio de datos en aplicaciones web y APIs (los servicios que usan las apps de tu celular para obtener datos de internet). Es más compacto que XML y muy fácil de leer y escribir para los programadores.

Utiliza dos estructuras básicas: objetos (pares clave-valor entre llaves {}) y arreglos (listas de valores entre corchetes []).

📖 Problemas Comunes de Compatibilidad

⚡ A. Problemas de Codificación de Caracteres (Encoding)

Las computadoras guardan texto como números. Para saber qué número corresponde a qué letra, usan una tabla de codificación. El problema surge cuando dos programas usan tablas distintas.

Codificaciones más importantes:

  • ASCII: La más antigua (1963). Solo contempla letras del inglés (128 caracteres). No incluye la ñ, los acentos (á, é, í, ó, ú) ni caracteres de otros idiomas.
  • ISO-8859-1 / Latin-1: Extensión de ASCII para idiomas occidentales. Incluye ñ y acentos. Muy usada en Argentina durante años.
  • UTF-8: El estándar moderno y universal. Puede representar más de 1 millón de caracteres: español, chino, árabe, emojis 😊, y más. Es la codificación recomendada hoy en día.

🧪 Ejemplo de error por encoding:

Un archivo generado en un sistema argentino antiguo (Latin-1) dice:

José Martínez — Dirección: Rincón 342

Si lo abrís con un programa configurado en UTF-8, puede aparecer como:

José Martínez — Dirección: Rincón 342 o con signos raros como:

Josß Martßnez

Los datos no se perdieron, solo se están "leyendo con el libro de códigos equivocado".

⚡ B. Problemas con Separadores (Delimitadores)

En archivos CSV, los datos se separan por un carácter especial llamado delimitador. El estándar internacional usa la coma (,), pero Argentina y muchos países de habla hispana usan el punto y coma (;) porque la coma ya se usa como separador decimal en los números (escribimos $1.500,50 en vez de $1,500.50).

⚠ Error muy común: Exportás una tabla de Excel a CSV y al abrirla en otro programa todos los datos aparecen en UNA SOLA columna. Causa: el archivo usa punto y coma pero el programa espera comas. Solución: revisar y especificar el delimitador correcto al importar.

⚡ C. Problemas con Formatos de Fecha y Número

No todos los países escriben las fechas igual, y eso genera conflictos al intercambiar datos:

País / Sistema Formato de fecha Ejemplo
🇦🇷 Argentina / Europa DD/MM/AAAA 15/03/2026 → 15 de marzo
Estados Unidos MM/DD/AAAA 03/15/2026 → 15 de marzo
🌐 ISO 8601 (Universal) AAAA-MM-DD 2026-03-15 → 15 de marzo

Si un sistema americano exporta y lo importa un sistema argentino, lo leerá como 1 de febrero cuando en realidad es 2 de enero. En contextos de salud, finanzas o logística, este tipo de error puede tener consecuencias graves.

⚡ D. Problemas de Fin de Línea (Line Endings)

Windows, Linux y Mac representan el "salto de línea" de manera diferente. Cuando se intercambian archivos de texto entre sistemas distintos, a veces aparecen caracteres extraños o todo el texto queda en una sola línea.

Sistema Operativo Carácter de fin de línea Código
Windows CR + LF (Carriage Return + Line Feed) \\r\\n
Linux / Mac moderno LF (Line Feed) \\n
Mac antiguo (pre-2001) CR (Carriage Return) \\r

📊 Conversión entre Formatos: ¿Qué se mantiene y qué se pierde?

Conversión Se conserva ✅ Se pierde ❌
Word (.docx) → TXT Todo el texto Formato, imágenes, tablas, colores, fuentes
Excel (.xlsx) → CSV Datos y texto de celdas Fórmulas, gráficos, colores, múltiples hojas
JSON → CSV Datos del nivel principal Datos anidados (sub-objetos), tipos de dato
XML → JSON Estructura y datos Atributos XML, comentarios, instrucciones
BMP → JPG (con compresión) Imagen visual aproximada Calidad original (pérdida irreversible)

⚠ Concepto clave — Compresión con y sin pérdida:

  • Sin pérdida (Lossless): PNG, ZIP, FLAC. El archivo reducido contiene TODA la información original. Al descomprimir, se recupera el archivo exacto.
  • Con pérdida (Lossy): JPG, MP3, AAC. Para reducir tamaño, descarta información que el ojo/oído difícilmente nota. Una vez comprimido así, la calidad original NO se puede recuperar, aunque se convierta a otro formato.

🖥 Guía Práctica — Conversión de Texto Plano a Planilla de Cálculo

GUÍA PASO A PASO: Importar CSV en Microsoft Excel

  1. Abrí Microsoft Excel. Andá a Datos → Obtener datos externos → Desde texto/CSV.
  2. Seleccioná el archivo .csv o .txt que querés importar.
  3. Excel mostrará una Vista Previa. Prestar atención al tipo de delimitador detectado.
  4. Si los datos aparecen en una sola columna, cambiá el separador de Coma a Punto y coma (o viceversa) hasta que las columnas se separen correctamente.
  5. Verificá la codificación de caracteres: si los acentos y la ñ se ven mal, cambiá de UTF-8 a ISO-8859-1 (Latin-1) o al revés.
  6. Verificá el formato de las columnas: columnas de fechas, números o texto deben marcarse correctamente para que Excel no las interprete mal.
  7. Hacé clic en Cargar o Importar.
